# 32 - グラフ領域のフォーマット

## 課題
グラフ領域のフォントサイズを +2 してください。さらに、グラフ領域の行を行単位にセル色を交互に変更する機能を追加してください。

## 実装内容

### 対象ファイル
- `src/PerformanceMonitorAnalyzer/MainWindow.xaml.cs`

### 変更箇所
1. `InitializeChart()` メソッド内で、ScottPlot v5 API を使用してフォントサイズを調整
2. 交互背景色（ゼブラストライプ）機能を実装

### 実装詳細

#### フォントサイズ調整

##### 軸ラベルのフォントサイズ
- X軸（時間）ラベル: デフォルト12px → 16px (+4) ※当初+2予定でしたが、可読性向上のため+4に調整
- Y軸（値）ラベル: デフォルト12px → 16px (+4)

```csharp
// 軸ラベルのフォントサイズ設定（デフォルト12 + 4 = 16）
PerformanceChart.Plot.Axes.Bottom.Label.FontSize = 16;
PerformanceChart.Plot.Axes.Left.Label.FontSize = 16;
```

##### 軸目盛りのフォントサイズ
- X軸目盛り: デフォルト10px → 16px (+6) ※当初+2予定でしたが、可読性向上のため+6に調整
- Y軸目盛り: デフォルト10px → 16px (+6)

```csharp
// 軸目盛りのフォントサイズ設定（デフォルト10 + 6 = 16）
PerformanceChart.Plot.Axes.Bottom.TickLabelStyle.FontSize = 16;
PerformanceChart.Plot.Axes.Left.TickLabelStyle.FontSize = 16;
```

##### 凡例のフォントサイズ
- 凡例: デフォルト11px → 13px (+2)

```csharp
// 凡例のフォントサイズ設定（デフォルト11 + 2 = 13）
PerformanceChart.Plot.Legend.FontSize = 13;
```

#### 交互背景色機能（ゼブラストライプ）

##### 概要
グラフのY軸領域を約10分割し、偶数番目の領域に薄いグレー色（#F8F8F8）の背景を適用することで、データの読み取りを容易にする機能を実装しました。

##### 実装メソッド

###### 1. SetupAlternatingGridBackground()
グリッド背景設定の初期化を行います。

```csharp
private void SetupAlternatingGridBackground()
{
    try
    {
        // グリッドの表示を有効にする
        PerformanceChart.Plot.Grid.MajorLineStyle.Width = 1;
        PerformanceChart.Plot.Grid.MajorLineColor = ScottPlot.Color.FromHex("#E0E0E0");
        
        // Y軸の主要グリッド線の間隔を設定
        PerformanceChart.Plot.Axes.Left.TickGenerator = new ScottPlot.TickGenerators.NumericAutomatic()
        {
            TargetTickCount = 10 // 約10本の主要グリッド線
        };
    }
    catch (Exception ex)
    {
        AppendToOperationLog($"グリッド背景設定エラー: {ex.Message}");
    }
}
```

###### 2. UpdateAlternatingBackground()
データプロット後に呼び出され、現在の軸範囲に基づいて交互背景色を動的に更新します。

```csharp
private void UpdateAlternatingBackground()
{
    try
    {
        // データがない場合は何もしない
        if (!_chartSeries.Any())
            return;

        // 既存の背景矩形を削除
        ClearAlternatingBackground();

        // 現在の軸範囲を取得
        var yMin = PerformanceChart.Plot.Axes.Left.Range.Min;
        var yMax = PerformanceChart.Plot.Axes.Left.Range.Max;
        var xMin = PerformanceChart.Plot.Axes.Bottom.Range.Min;
        var xMax = PerformanceChart.Plot.Axes.Bottom.Range.Max;
        
        // Y軸のグリッド間隔を計算（約10分割）
        var yRange = yMax - yMin;
        var gridSpacing = yRange / 10.0;
        
        // 交互の背景色を追加
        var lightGray = ScottPlot.Color.FromHex("#F8F8F8");
        
        for (int i = 0; i < 10; i += 2)
        {
            var y1 = yMin + (i * gridSpacing);
            var y2 = yMin + ((i + 1) * gridSpacing);
            
            // 範囲チェック
            if (y2 > yMax) y2 = yMax;
            if (y1 >= yMax) break;
            
            var rect = PerformanceChart.Plot.Add.Rectangle(xMin, y1, xMax, y2);
            rect.FillStyle.Color = lightGray;
            rect.LineStyle.Width = 0; // 境界線なし
            
            // 背景として識別できるようにタグを設定
            if (!_backgroundRectangles.Contains(rect))
            {
                _backgroundRectangles.Add(rect);
            }
        }
    }
    catch (Exception ex)
    {
        AppendToOperationLog($"交互背景色更新エラー: {ex.Message}");
    }
}
```

###### 3. ClearAlternatingBackground()
既存の交互背景色を削除します。

```csharp
private void ClearAlternatingBackground()
{
    try
    {
        foreach (var rect in _backgroundRectangles)
        {
            PerformanceChart.Plot.Remove(rect);
        }
        _backgroundRectangles.Clear();
    }
    catch (Exception ex)
    {
        AppendToOperationLog($"背景色削除エラー: {ex.Message}");
    }
}
```

##### 追加フィールド
背景矩形を追跡するためのフィールドを追加しました。

```csharp
// 交互背景色の矩形を追跡
private readonly List<ScottPlot.Plottables.Rectangle> _backgroundRectangles = new();
```

##### 統合部分
既存のデータプロット処理に背景色更新を統合しました。

```csharp
// AddCounterToChart() および AddCounterToChartInternal() メソッド内
PerformanceChart.Plot.Axes.AutoScale();

// 交互背景色を更新
UpdateAlternatingBackground();

PerformanceChart.Refresh();
```

## 技術仕様

### 使用ライブラリ
- ScottPlot.WPF v5.0.41

### API詳細
ScottPlot v5では、以下のAPIを使用しています：

- **軸ラベル**: `Plot.Axes.[Bottom|Left].Label.FontSize`
- **軸目盛り**: `Plot.Axes.[Bottom|Left].TickLabelStyle.FontSize`
- **凡例**: `Plot.Legend.FontSize`
- **矩形描画**: `Plot.Add.Rectangle(xMin, yMin, xMax, yMax)`
- **色指定**: `ScottPlot.Color.FromHex("#F8F8F8")`
- **軸範囲取得**: `Plot.Axes.[Bottom|Left].Range.[Min|Max]`

### 色仕様
- **交互背景色**: #F8F8F8 （薄いグレー）
- **グリッド線色**: #E0E0E0 （ライトグレー）

## 影響範囲

この変更により、以下の要素が向上します：

1. **フォントサイズ向上**
   - X軸（時間軸）の軸ラベルと目盛り表示
   - Y軸（値軸）の軸ラベルと目盛り表示
   - 凡例の表示

2. **視認性向上**
   - Y軸方向の交互背景色により、データの読み取りが容易
   - グリッドラインによる値の判読性向上
   - 10分割による適度な視覚的区切り

3. **動的更新**
   - データプロット時の自動背景更新
   - 軸範囲変更時の背景再描画
   - グラフクリア時の背景削除

## 注意事項

- この変更はグラフが初期化される際に適用されます
- WPFアプリケーションのため、Windows環境でのみ動作確認可能
- ScottPlot v5のAPIを使用しているため、バージョン互換性に注意が必要
- 背景矩形の数が多い場合、パフォーマンスに影響を与える可能性があります
- 交互背景色は軸範囲に基づいて動的に計算されるため、データ範囲によって表示が変わります
