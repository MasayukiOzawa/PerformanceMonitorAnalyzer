# 13-グラフ表示機能の実装 - .NET 8.0互換性対応

## 概要
Performance Monitor Analyzer にグラフ表示機能を実装しました。当初LiveCharts.Wpfを使用しましたが、.NET 8.0との互換性問題が発生したため、ScottPlot.WPFに移行しました。これにより、データテーブルに読み込んだパフォーマンスカウンターデータを時系列グラフとして視覚的に分析できるようになりました。

## 実装の経緯と問題解決

### 初期実装（LiveCharts.Wpf 0.9.7）
- 時系列データのグラフ表示機能を実装
- 複数カウンターの同時表示
- リアルタイム連携機能

### 互換性問題の発生
LiveCharts 0.9.7が.NET Framework専用のライブラリであり、.NET 8.0プロジェクトで以下のエラーが発生：

```
error MC3050: 型 'TimeUnitType' が見つかりません。型名では大文字と小文字が区別されることに注意してください。
warning NU1701: パッケージ 'LiveCharts.Wpf 0.9.7' はプロジェクトのターゲット フレームワーク 'net8.0-windows7.0' ではなく '.NETFramework,Version=v4.6.1...' を使用して復元されました。
```

### 解決方法：ScottPlot.WPFへの移行

#### パッケージの変更
```xml
<!-- 変更前 -->
<PackageReference Include="LiveCharts.Wpf" Version="0.9.7" />

<!-- 変更後 -->
<PackageReference Include="ScottPlot.WPF" Version="5.0.41" />
```

{
    PerformanceChart.Plot.Clear();
    PerformanceChart.Plot.XLabel("時間");
    PerformanceChart.Plot.YLabel("値");
    PerformanceChart.Plot.Title("パフォーマンスカウンター");
    
    PerformanceChart.Plot.Axes.DateTimeTicksBottom();
    PerformanceChart.Refresh();
}

##### データ追加・削除メソッド
```csharp
private void AddCounterToChart(string counter)
{
    var dataPoints = _counterData[counter];
    var xValues = dataPoints.Select(dp => dp.Timestamp.ToOADate()).ToArray();
    var yValues = dataPoints.Select(dp => dp.Value).ToArray();
    
    var scatter = PerformanceChart.Plot.Add.Scatter(xValues, yValues);
    scatter.LegendText = GetCounterDisplayName(counter);
    scatter.LineWidth = 2;
    scatter.MarkerSize = 0;
    
    _chartSeries[counter] = scatter;
    PerformanceChart.Plot.Axes.AutoScale();
    PerformanceChart.Refresh();
}

private void RemoveCounterFromChart(string counter)
{
    if (_chartSeries.TryGetValue(counter, out var scatter))
    {
        PerformanceChart.Plot.Remove(scatter);
        _chartSeries.Remove(counter);
        PerformanceChart.Refresh();
    }
}
```

## 技術仕様

### グラフの特徴
- **時系列表示**: X軸が時間、Y軸がパフォーマンス値
- **複数カウンター対応**: 異なる色の線で複数カウンターを同時表示
- **高パフォーマンス**: マーカー非表示で高速描画
- **自動スケール**: データに応じて軸が自動調整

### ScottPlotの利点
1. **.NET 8.0完全サポート**: 互換性問題が解決
2. **高性能**: 大量のデータポイントでも高速描画
3. **シンプルなAPI**: 直感的で使いやすいAPI
4. **豊富な機能**: 時系列データに特化した機能
5. **アクティブな開発**: 継続的にアップデートされている

### データフォーマット
- **X軸**: DateTime.ToOADate() を使用した時間軸
- **Y軸**: double 型の数値データ
- **時間軸**: 自動的に日時フォーマットで表示

### UI連携
- チェックボックス選択 → 実行ボタン → データ読み込み → グラフ更新
- チェックボックス解除 → 即座にグラフから削除
- データがない場合は説明メッセージを表示

現在Linuxベースの開発環境ではWPFアプリケーションのビルド・実行ができないため、Windows環境での手動テストが必要です：

1. Windows環境で `dotnet build` 実行
2. `dotnet run` でアプリケーション起動
3. サンプルBLGファイルまたは実際のBLGファイルを読み込み
4. カウンター選択→実行→グラフ表示を確認

## 使用方法

1. BLGファイルを読み込み（relog.exe方式推奨）
2. 左側のカウンター一覧でチェックボックスを選択
3. 「選択されたカウンターを読み込み」ボタンをクリック
4. 右側のグラフエリアに時系列グラフが表示される
5. チェックボックスを外すと該当するグラフが削除される
