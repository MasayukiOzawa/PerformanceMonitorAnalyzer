# 13-グラフ表示機能の実装

## 概要
Performance Monitor Analyzer にグラフ表示機能を実装しました。これにより、データテーブルに読み込んだパフォーマンスカウンターデータを時系列グラフとして視覚的に分析できるようになりました。

## 実装内容

### 1. ライブラリの追加
- **LiveCharts.Wpf 0.9.7** をプロジェクトに追加
- WPFアプリケーションで高パフォーマンスなチャート表示を実現

### 2. XAMLの更新
- `MainWindow.xaml` にLiveCharts名前空間を追加
- グラフ表示エリアに `CartesianChart` コンポーネントを配置
- X軸（時間）、Y軸（値）の設定
- グラフデータがない時のメッセージ表示UI

### 3. C#コードの実装

#### MainWindow.xaml.cs の主要変更
- LiveCharts関連のusingステートメント追加
- `SeriesCollection`、`DateTimeFormatter`、`ValueFormatter` プロパティ追加
- `InitializeChart()` メソッドで初期設定
- `AddCounterToChart()` メソッド実装: データをグラフに追加
- `RemoveCounterFromChart()` メソッド実装: データをグラフから削除
- `UpdateChartVisibility()` メソッド: グラフの表示状態管理

### 4. データ連携の実装
- 既存の `PerformanceDataPoint` クラスをそのまま活用
- relog.exe によるデータ読み込み完了時に自動的にグラフ更新
- チェックボックスの状態とグラフ表示が連動

## 技術仕様

### グラフの特徴
- **時系列表示**: X軸が時間、Y軸がパフォーマンス値
- **複数カウンター対応**: 異なる色の線で複数カウンターを同時表示
- **インタラクティブ**: 凡例クリックで個別表示/非表示切り替え
- **パフォーマンス最適化**: ポイント非表示、直線描画で高速化

### データフォーマット
- **X軸**: DateTime.Ticks を使用した時間軸
- **Y軸**: double 型の数値データ
- **フォーマッター**: 時間は "MM/dd HH:mm:ss"、値は "N2" 形式

### UI連携
- チェックボックス選択 → 実行ボタン → データ読み込み → グラフ更新
- チェックボックス解除 → 即座にグラフから削除
- データがない場合は説明メッセージを表示

## 実装のポイント

### 最小限の変更
- 既存のデータ処理ロジックは一切変更せず
- 既存の `AddCounterToChart()` / `RemoveCounterFromChart()` の空実装を活用
- データテーブル機能とグラフ機能の並列稼働

### エラーハンドリング
- データが存在しないカウンターの処理
- 重複シリーズの防止
- UI更新時の例外処理

### パフォーマンス考慮
- ポイントジオメトリを無効化（`PointGeometry = null`）
- スムージングを無効化（`LineSmoothness = 0`）
- アニメーション時間を短縮（`AnimationsSpeed="0:0:0.5"`）

## 使用方法

1. BLGファイルを読み込み（relog.exe方式推奨）
2. 左側のカウンター一覧でチェックボックスを選択
3. 「選択されたカウンターを読み込み」ボタンをクリック
4. 右側のグラフエリアに時系列グラフが表示される
5. チェックボックスを外すと該当するグラフが削除される

## 今後の改善点

1. **グラフの時間範囲指定**: 時間スライダーとグラフの連動
2. **Y軸の自動スケーリング**: カウンターの単位に応じた軸設定
3. **グラフのエクスポート**: PNG/JPG形式での保存機能
4. **ズーム機能**: 特定時間範囲の詳細表示
5. **複数Y軸対応**: 単位の異なるカウンターの同時表示

## テスト方法

現在Linuxベースの開発環境ではWPFアプリケーションのビルド・実行ができないため、Windows環境での手動テストが必要です：

1. Windows環境で `dotnet build` 実行
2. `dotnet run` でアプリケーション起動
3. サンプルBLGファイルまたは実際のBLGファイルを読み込み
4. カウンター選択→実行→グラフ表示を確認
