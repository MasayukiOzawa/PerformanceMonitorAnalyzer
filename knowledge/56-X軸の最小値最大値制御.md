# X軸の最小値/最大値制御機能の実装

## 課題の概要

**Issue #56**: X軸の最小値/最大値を選択している時間軸の範囲の値として設定し、スクロールして縮小/拡大した際にも時間軸の範囲外を表示しないようにする機能を実装。

## 実装内容

### 1. 問題の分析

#### 既存の状況
- 時間範囲選択機能（StartTimeSlider, EndTimeSlider）は既に実装済み
- ScottPlotライブラリを使用してグラフを表示
- 現在は`AutoScale()`で全データの範囲を表示
- 時間スライダーの値はデータ読み込み時にのみ使用され、グラフ表示には反映されていない

#### 解決が必要な問題
- グラフのX軸範囲が時間選択に連動していない
- ズーム/スクロール時に選択範囲外のデータが表示される

### 2. 実装したメソッド

#### 2.1 `GetSelectedTimeRange()` - 時間範囲計算メソッド
```csharp
private (DateTime startTime, DateTime endTime) GetSelectedTimeRange()
```

**役割**: 時間スライダーの値（0-100%）から実際のDateTime値を計算

**動作**:
- 時間範囲が検出されていない場合は、ファイル全体の時間範囲を返す
- スライダーの値（パーセンテージ）を基に、ファイルの時間範囲内での実際の開始/終了時刻を計算

#### 2.2 `UpdateChartXAxisRange()` - X軸範囲設定メソッド
```csharp
private void UpdateChartXAxisRange()
```

**役割**: グラフのX軸範囲を選択された時間範囲に設定

**動作**:
- `GetSelectedTimeRange()`で計算された時間範囲を取得
- ScottPlotの`PerformanceChart.Plot.Axes.Bottom.Min/Max`にOADate形式で設定
- グラフを更新（`PerformanceChart.Refresh()`）
- デバッグログ出力

### 3. 修正したメソッド

#### 3.1 `TimeSlider_ValueChanged()` - スライダー変更イベント
**追加内容**: 
```csharp
// グラフのX軸範囲を更新
UpdateChartXAxisRange();
```

**効果**: 時間スライダーを操作すると即座にグラフのX軸範囲が更新される

#### 3.2 `RefreshChartWithCurrentType()` - グラフ再描画メソッド
**追加内容**:
```csharp
// X軸の範囲を選択された時間範囲に設定
UpdateChartXAxisRange();
```

**効果**: グラフタイプ切り替え時やデータ更新時にもX軸範囲が適用される

#### 3.3 `AddLineChartSeries()` - 折れ線グラフシリーズ追加
**追加内容**:
```csharp
// X軸の範囲を選択された時間範囲に設定
UpdateChartXAxisRange();
```

**効果**: 新しいカウンターをグラフに追加した際にもX軸範囲が適用される

#### 3.4 `AddCounterToChartInternal()` - 内部カウンター追加
**追加内容**:
```csharp
// X軸の範囲を選択された時間範囲に設定
UpdateChartXAxisRange();
```

**効果**: チェックボックス経由でカウンターを追加した際にもX軸範囲が適用される

#### 3.5 `UpdateTimeRangeUI()` - 時間範囲UI更新
**追加内容**:
```csharp
// グラフのX軸範囲を初期化
UpdateChartXAxisRange();
```

**効果**: BLGファイル読み込み後の初期状態でもX軸範囲が適用される

#### 3.6 `RefreshCounterInChart()` - 個別カウンター更新
**追加内容**:
```csharp
// X軸の範囲を選択された時間範囲に設定
UpdateChartXAxisRange();
```

**効果**: スケール変更等でカウンターを個別更新した際にもX軸範囲が維持される

### 4. 技術的詳細

#### 4.1 ScottPlotでの時間軸制御
- `PerformanceChart.Plot.Axes.Bottom.Min/Max`プロパティでX軸範囲を制御
- DateTime値は`ToOADate()`でOLE Automation Date形式に変換
- `DateTimeTicksBottom()`で時間軸目盛りを設定済み

#### 4.2 時間計算ロジック
```csharp
var totalDuration = _fileEndTime - _fileStartTime;
var startTime = _fileStartTime.AddMilliseconds(totalDuration.TotalMilliseconds * StartTimeSlider.Value / 100);
var endTime = _fileStartTime.AddMilliseconds(totalDuration.TotalMilliseconds * EndTimeSlider.Value / 100);
```

- ファイル全体の時間範囲を100%として分割
- スライダーの値（0-100）をパーセンテージとして使用
- ミリ秒単位で精密な時間計算

### 5. 動作フロー

1. **BLGファイル読み込み**
   - 時間範囲検出 → `UpdateTimeRangeUI()` → X軸範囲初期化

2. **時間スライダー操作**
   - `TimeSlider_ValueChanged()` → X軸範囲即座に更新

3. **カウンター選択**
   - チェックボックス → `AddCounterToChartInternal()` → X軸範囲適用
   - 直接追加 → `AddLineChartSeries()` → X軸範囲適用

4. **グラフタイプ切り替え**
   - `RefreshChartWithCurrentType()` → X軸範囲再適用

5. **スケール調整**
   - `RefreshCounterInChart()` → X軸範囲維持

### 6. 期待される効果

#### ユーザビリティの向上
- 時間スライダーとグラフ表示が連動
- 選択した時間範囲のみにフォーカスした分析が可能
- 不要な範囲外データが非表示になることで視認性向上

#### パフォーマンスの向上
- 表示範囲を限定することでグラフ描画が高速化（データポイント数は変わらないが表示範囲が限定される）

#### 分析の精度向上
- 特定時間帯のパフォーマンス変動を詳細に分析可能
- 時間軸のズーム機能と組み合わせて精密な分析が可能

### 7. エラーハンドリング

- 時間範囲が検出されていない場合は処理をスキップ
- X軸範囲設定でエラーが発生した場合はログに記録
- 計算エラーや不正な値の場合はデフォルト値を使用

### 8. 実装時の考慮事項

#### 最小限の変更
- 既存のコード構造を最大限活用
- 新しいメソッドは最小限に留める
- 既存メソッドへの変更も必要最小限

#### 一貫性の保持
- 全てのグラフ更新パスでX軸範囲が適用されるよう徹底
- スライダー操作、データ更新、グラフタイプ切り替え等、全ての場面で一貫した動作

#### パフォーマンス配慮
- `UpdateChartXAxisRange()`は軽量な処理
- 必要以上の再描画を避ける設計

## 実装成果

この実装により、Performance Monitor Analyzerの時間軸制御機能が大幅に向上し、ユーザーが選択した時間範囲に応じてグラフ表示が動的に調整されるようになりました。これにより、パフォーマンス分析の精度とユーザビリティが向上します。
