# 66-ウィンドウサイズ表示機能の実装

## タスク概要

画面右下のステータスバー領域にウィンドウサイズとグラフサイズを表示し、手動で設定できる機能を実装しました。

## 要求仕様

### 初期要求
- 画面右下のステータスバー領域としてウィンドウサイズを表示
- ウィンドウサイズを手動設定できる機能

### 追加要求（ユーザーコメント）
- グラフ表示エリアのサイズ表示と、サイズ変更もウィンドウサイズと同じ操作感で実施したい

## 実装アプローチ

### 1. ステータスバーの追加（第1段階）

**MainWindow.xaml の変更:**
- 既存のGrid構造に新しいRowDefinition（Auto）を追加
- 最下部（Grid.Row="5"）にStatusBarを配置
- 3列レイアウトを使用（ステータステキスト、セパレーター、ウィンドウサイズ）

```xml
<!-- ステータスバー -->
<StatusBar Grid.Row="5" Background="LightGray" BorderBrush="Gray" BorderThickness="0,1,0,0">
    <!-- 3列レイアウト: ステータステキスト、セパレーター、ウィンドウサイズ -->
</StatusBar>
```

### 2. ウィンドウサイズ表示機能

**MainWindow.xaml.cs の変更:**
- `InitializeWindowSizeTracking()` メソッドを追加
- `SizeChanged` および `StateChanged` イベントハンドラーを実装
- リアルタイムでウィンドウサイズとステート（最大化/最小化）を表示

```csharp
private void UpdateWindowSizeDisplay()
{
    var width = this.ActualWidth > 0 ? this.ActualWidth : this.Width;
    var height = this.ActualHeight > 0 ? this.ActualHeight : this.Height;
    
    string stateText = this.WindowState switch
    {
        WindowState.Maximized => " (最大化)",
        WindowState.Minimized => " (最小化)",
        _ => ""
    };
    
    WindowSizeText.Text = $"ウィンドウサイズ: {width:F0}×{height:F0}{stateText}";
}
```

### 3. 手動サイズ設定ダイアログ（第1段階）

**新規ファイル作成:**
- `WindowSizeSettingDialog.xaml` - ダイアログのUI
- `WindowSizeSettingDialog.xaml.cs` - ダイアログのロジック

**主要機能:**
- 現在のウィンドウサイズ・状態の表示
- ウィンドウ状態選択（通常/最大化/最小化）
- カスタムサイズ入力（幅・高さ）
- プリセットサイズボタン（1024×768、1280×720、1366×768、1920×1080）
- リアルタイムプレビュー
- 入力値検証（幅400-3840、高さ300-2160）

### 4. グラフサイズ表示機能の追加（第2段階）

**ステータスバーの拡張:**
- 4列レイアウトに変更（ステータステキスト、グラフサイズ、セパレーター、ウィンドウサイズ）
- リアルタイムでグラフサイズを表示：`グラフサイズ: 800×400`

**MainWindow.xaml.cs の変更:**
- `InitializeGraphSizeTracking()` メソッドを追加
- `PerformanceChart.SizeChanged` イベントハンドラーを実装
- リアルタイムでグラフサイズを表示

```csharp
private void UpdateGraphSizeDisplay()
{
    var width = PerformanceChart.ActualWidth > 0 ? PerformanceChart.ActualWidth : PerformanceChart.Width;
    var height = PerformanceChart.ActualHeight > 0 ? PerformanceChart.ActualHeight : PerformanceChart.Height;
    
    // NaNやInfinityをチェック
    if (double.IsNaN(width) || double.IsInfinity(width) || width <= 0)
        width = 800; // デフォルト値
    if (double.IsNaN(height) || double.IsInfinity(height) || height <= 0)
        height = 400; // デフォルト値
    
    GraphSizeText.Text = $"グラフサイズ: {width:F0}×{height:F0}";
}
```

### 5. グラフサイズ設定ダイアログ（第2段階）

**新規ファイル作成:**

**主要機能:**

- プリセットサイズボタン（800×400、1000×500、1200×600、1400×700）
- リアルタイムプレビュー
- 入力値検証（幅200-3840、高さ150-2160）

## 最小限変更の原則

### 既存機能への影響なし
- 既存のレイアウト構造を維持
- 新しいRowDefinitionの追加のみ（ウィンドウサイズ機能）
- 新しいColumnDefinitionの追加のみ（グラフサイズ機能）
- 既存のイベントハンドラーに干渉なし

### コードの分離
- ウィンドウサイズ関連機能を独立した`#region`ブロックで分離
- グラフサイズ関連機能を独立した`#region`ブロックで分離
- 新規ダイアログクラスとして分離実装
- 既存クラスの変更を最小限に抑制

## 動作フロー

### ウィンドウサイズ関連
1. **アプリケーション起動時**
   - `InitializeWindowSizeTracking()` でイベントハンドラー登録
   - 初期ウィンドウサイズを表示

2. **ウィンドウサイズ変更時**
   - `SizeChanged`/`StateChanged` イベント発火
   - `UpdateWindowSizeDisplay()` で表示更新

3. **手動設定時**
   - サイズ表示クリック → `ShowWindowSizeSettingDialog()` 実行
   - ダイアログで設定値入力・検証
   - OK時にウィンドウサイズ・状態を適用

### グラフサイズ関連
1. **アプリケーション起動時**
   - `InitializeGraphSizeTracking()` でイベントハンドラー登録
   - 初期グラフサイズを表示

2. **グラフサイズ変更時**
   - `PerformanceChart.SizeChanged` イベント発火
   - `UpdateGraphSizeDisplay()` で表示更新

3. **手動設定時**
   - サイズ表示クリック → `ShowGraphSizeSettingDialog()` 実行
   - ダイアログで設定値入力・検証
   - OK時にグラフエリアサイズを調整

## 後続調整: ウィンドウサイズ設定ダイアログの見切れ対策

- `WindowSizeSettingDialog.xaml` の固定 `Height="310"` が、テーマ余白や表示倍率次第で下部ボタン行を収めきれず、`適用` / `キャンセル` が見切れることがあった。
- 対策として、ダイアログを `SizeToContent="Height"` に変更し、`MinHeight` / `MinWidth` を持たせて内容量に応じて十分なサイズを確保するようにした。
- あわせて、プレビュー表示とボタン群を別行へ分離し、最下段が常に見えるレイアウトへ整理した。
- `GraphSizeSettingDialog.xaml` も同様に固定高さをやめ、`WrapPanel` によるプリセットボタン折り返しと、プレビュー / ボタン行の分離で見切れにくいレイアウトへ揃えた。
- さらに両ダイアログの `適用` ボタンは `SecondaryActionButtonStyle` + `MinWidth="120"` に統一し、メイン画面の `パターンを適用` ボタンと同じトーンで視認しやすい見た目へ揃えた。
