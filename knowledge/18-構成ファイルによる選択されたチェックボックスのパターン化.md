# 18-構成ファイルによる選択されたチェックボックスのパターン化

## タスク概要

YAMLフォーマットの構成ファイルで、カウンターのチェックボックスが選択された状態をパターン化して登録できる機能を実装しました。

## 実装内容

### 1. YAMLライブラリの追加
- **パッケージ**: YamlDotNet 15.1.1
- **目的**: YAML設定ファイルの読み書き
- **設定場所**: PerformanceMonitorAnalyzer.csproj

### 2. データ構造の実装
新しいファイル: `src/PerformanceMonitorAnalyzer/CounterPattern.cs`

#### 主要クラス
- **CounterPatternConfig**: YAML設定ファイルのルートオブジェクト
- **CounterPattern**: 個別パターンの定義（名前、説明、カウンターリスト）
- **CounterDefinition**: 個別カウンターの設定（名前、スケール、有効状態）
- **CounterPatternManager**: パターン管理の中核クラス

#### 機能
- YAML設定ファイルの読み込み・解析
- デフォルト設定ファイルの自動作成
- パターンの検索・取得
- エラーログ機能

### 3. UI拡張

#### メニューバー拡張
- **新規メニュー**: 「パターン(_P)」
  - カウンターパターンを適用: 動的に生成されるサブメニュー
  - パターン設定ファイルを開く: 設定ファイルの直接編集
  - パターン設定を再読み込み: 設定変更の反映

#### カウンター選択パネル拡張
- **パターン選択コンボボックス**: 利用可能パターンの表示
- **パターン適用ボタン**: 選択パターンの一括適用
- **ツールチップ**: 各パターンの説明表示

### 4. パターン適用ロジック

#### 検索アルゴリズム
1. **完全一致検索**: カウンター名の厳密マッチング
2. **ワイルドカードパターン**: `*`（任意文字列）、`?`（単一文字）対応
3. **正規表現変換**: ワイルドカードを正規表現に自動変換
4. **フォールバック**: 正規表現エラー時の部分文字列マッチ

#### 適用処理
- 既存選択の全解除
- パターンに含まれる各カウンターの検索・選択
- スケール値の自動設定
- 適用結果のフィードバック表示

### 5. サンプル設定ファイル
ファイル: `config/counter-patterns.yaml`

#### 定義済みパターン（6種類）
1. **基本システム監視**: CPU、メモリ、ディスクの基本項目
2. **ネットワーク監視**: ネットワークトラフィック関連
3. **詳細システム監視**: 詳細なシステム分析項目
4. **SQLサーバー監視**: SQL Server特有のパフォーマンス項目
5. **Webサーバー監視**: IIS関連の監視項目
6. **高負荷診断**: システム高負荷時の詳細診断

## 技術的特徴

### 非同期処理
- パターン設定の読み込み: `InitializePatternManagerAsync()`
- カウンター検索: `FindCounterNodeAsync()`, `FindCountersByPatternAsync()`
- パターン適用: `ApplyCounterPatternAsync()`

### エラーハンドリング
- 設定ファイル読み込みエラーの詳細ログ
- パターン適用時の個別エラー処理
- 未検出カウンターの明確なフィードバック

### スケール機能統合
- パターンごとのカウンタースケール設定
- 既存のスケール管理システムとの統合
- デフォルト値（1.0）の自動設定

## YAML設定ファイル構造

```yaml
patterns:
  パターン名:
    name: 表示名
    description: 説明文
    counters:
      - name: \カウンター\パス
        scale: 1.0
        enabled: true
```

## 使用方法

### 基本的な使用手順
1. BLGファイルを読み込み
2. パターン選択コンボボックスから目的のパターンを選択
3. 「パターンを適用」ボタンをクリック
4. 自動選択されたカウンターでデータ分析開始

### カスタマイズ
1. 「パターン」→「パターン設定ファイルを開く」
2. YAML形式で新しいパターンを追加・編集
3. 「パターン設定を再読み込み」で変更を反映

## 実装時の課題と解決策

### 課題1: Linux環境でのWPFビルドエラー
- **問題**: GitHub Codespaces（Linux）でWPFアプリケーションがビルドできない
- **対応**: ソースコード変更のみに集中、Windows環境での動作は想定通り

### 課題2: 既存UIレイアウトへの影響
- **問題**: パターン選択UIの追加によるレイアウト調整
- **解決**: グリッド行定義の追加、既存要素の行番号調整

### 課題3: パターンマッチングの柔軟性
- **問題**: カウンター名の表記ゆれや環境差異
- **解決**: 完全一致→ワイルドカード→部分一致の段階的検索
