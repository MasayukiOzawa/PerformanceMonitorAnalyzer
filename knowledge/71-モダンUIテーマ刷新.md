# モダン UI テーマ刷新

## 概要

既存の `MainWindow.xaml.cs` の処理や各種イベントを維持したまま、WPF 標準コントロールの見た目を共通テーマで統一し、メイン画面と設定ダイアログをカードベースのモダン UI に刷新した。

## 実施内容

### 1. テーマ基盤の追加
- `src\PerformanceMonitorAnalyzer\App.xaml` から共通 `ResourceDictionary` を読み込む構成へ変更
- `src\PerformanceMonitorAnalyzer\Styles\Colors.xaml`
  - アプリ全体の背景色、サーフェス色、アクセント色、成功 / 警告 / 危険色、境界線色を定義
- `src\PerformanceMonitorAnalyzer\Styles\Controls.xaml`
  - `Button`, `GroupBox`, `Expander`, `TabControl`, `DataGrid`, `StatusBar` などの共通スタイルを定義

### 2. メイン画面の見た目刷新
- メニューバーをカード風コンテナに載せ、右側にアプリ名を表示
- プログレス表示、左カウンターパネル、グラフ領域、凡例、スケール設定、下部データテーブル / ログ領域を淡色トーンで統一
- トグルボタンや主要アクションボタンに共通スタイルを適用
- ファイル情報、空状態メッセージ、データテーブル説明ヘッダーなどの配色を整理

### 2.1. レビュー対応: ExecuteButton の前景色継承修正
- `src\PerformanceMonitorAnalyzer\MainWindow.xaml` の「🚀 選択されたカウンターを読み込み」ボタン内 `TextBlock` で指定していた `Foreground="White"` を削除し、祖先 `Button` の `Foreground` にバインドする形へ変更した。
- これにより、有効時は `PrimaryActionButtonStyle` が設定する白文字を維持しつつ、無効時は `src\PerformanceMonitorAnalyzer\Styles\Controls.xaml` の `IsEnabled=False` トリガーが設定する `TextMutedBrush` をそのまま反映できるようになった。

### 2.2. レビュー対応: TreeView 選択文字色のアクティブ/非アクティブ切り分け
- `src\PerformanceMonitorAnalyzer\MainWindow.xaml` のカウンターツリー表示名 / 件数表示の `TextBlock.Style` で、`TreeViewItem.IsSelected=True` のときに常に `White` を設定していた `DataTrigger` を見直した。
- `Selector.IsSelectionActive` を条件に追加した `MultiDataTrigger` へ変更し、アクティブ選択時は `{DynamicResource {x:Static SystemColors.HighlightTextBrushKey}}`、非アクティブ選択時は `{DynamicResource {x:Static SystemColors.InactiveSelectionHighlightTextBrushKey}}` を使うようにした。
- これにより、非アクティブ選択時でも淡い背景に対して読みやすいシステム既定の文字色が使われ、未選択時は従来どおり `TextColor` / `TextMutedBrush` を維持できる。

### 3. ダイアログの見た目統一
- `GraphSizeSettingDialog.xaml`
- `WindowSizeSettingDialog.xaml`

上記 2 つのダイアログを共通カードスタイルへ合わせ、プライマリアクション / 補助アクション / プリセットボタンの見た目を整理した。

## 検証

### ビルド
既存の既定出力先では実行中の `PerformanceMonitorAnalyzer.exe` によるロックが発生しやすいため、検証時は外部出力先を指定してビルドした。

```powershell
dotnet build --nologo -v q -p:OutputPath=%TEMP%\pma-isolated-output\ src\PerformanceMonitorAnalyzer\PerformanceMonitorAnalyzer.csproj
```

### テスト
`dotnet test` は既定の自己完結型 / apphost のまま外部出力へ切り替えると testhost のランタイム解決で失敗したため、検証時は framework-dependent + `UseAppHost=false` で実行した。

```powershell
dotnet test --nologo -v normal -p:OutputPath=%TEMP%\pma-isolated-test-fdd\ -p:SelfContained=false -p:UseAppHost=false tests\PerformanceMonitorAnalyzer.Tests\PerformanceMonitorAnalyzer.Tests.csproj
```
