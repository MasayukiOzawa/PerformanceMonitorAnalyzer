# ログ出力領域の追加 (Issue #24)

## 概要

PerformanceMonitorAnalyzerアプリケーションにおいて、従来メッセージボックスで表示していた情報を、ログ出力領域で確認できるよう機能を追加しました。

## 要求仕様

- **対象メッセージ**: blgファイル読み込み、パターン適用、カウンター読み込み
- **保持対象**: エラー発生時のメッセージボックスは維持
- **UI要件**: メニューバーを使わずに情報確認可能
- **追加要件**: error.logファイル内容の確認機能

## 実装内容

### 1. 新しいクラス定義

#### LogEntry クラス
```csharp
public class LogEntry : INotifyPropertyChanged
{
    public DateTime Timestamp { get; set; }
    public LogLevel Level { get; set; }
    public string Message { get; set; } = string.Empty;
    public string FormattedTimestamp => Timestamp.ToString("yyyy-MM-dd HH:mm:ss");
    public string LevelDisplay => Level.ToString();
    public System.Windows.Media.Brush TextColor => Level switch
    {
        LogLevel.Error => System.Windows.Media.Brushes.Red,
        LogLevel.Warning => System.Windows.Media.Brushes.Orange,
        LogLevel.Info => System.Windows.Media.Brushes.Blue,
        LogLevel.Success => System.Windows.Media.Brushes.Green,
        _ => System.Windows.Media.Brushes.Black
    };
}
```

#### LogLevel 列挙体
```csharp
public enum LogLevel
{
    Info,
    Success,
    Warning,
    Error
}
```

### 2. ログ管理機能

#### フィールド追加
```csharp
private readonly ObservableCollection<LogEntry> _operationLogs = new();
private readonly ObservableCollection<LogEntry> _errorLogs = new();
```

#### 主要メソッド
- `AddOperationLog(LogLevel level, string message)`: 操作ログ追加
- `LoadErrorLogFromFile()`: error.logファイル読み込み
- `ParseLogLine(string line)`: ログ行解析
- `ClearOperationLogs()`: 操作ログクリア
- `ClearErrorLogs()`: エラーログクリア

### 3. UI実装

#### UI構造の変更 (フィードバック対応)
**初期実装**: データテーブルタブ内にログタブを配置
```xml
<TabControl x:Name="DataTabControl">
  <!-- データタブ + ログタブが混在 -->
</TabControl>
```

**改善後**: データテーブルとログ表示を同レベルに分離
```xml
<Grid>
  <Grid.ColumnDefinitions>
    <ColumnDefinition Width="*"/>                  <!-- データテーブル -->
    <ColumnDefinition Width="5"/>                  <!-- スプリッター -->
    <ColumnDefinition Width="350" MinWidth="250"/> <!-- ログ表示 -->
  </Grid.ColumnDefinitions>
  
  <!-- 左側: データテーブル -->
  <GroupBox Grid.Column="0" Header="📊 データテーブル">
    <TabControl x:Name="DataTabControl">
      <!-- データタブのみ -->
    </TabControl>
  </GroupBox>
  
  <!-- 右側: ログ表示 -->
  <GroupBox Grid.Column="2" Header="📋 ログ表示">
    <TabControl x:Name="LogTabControl">
      <!-- ログタブのみ -->
    </TabControl>
  </GroupBox>
</Grid>
```

#### 操作ログタブ
- **ヘッダー**: "📋 操作ログ"
- **機能**: ログクリアボタン
- **表示**: DataGrid（時間、レベル、メッセージ）
- **色分け**: ログレベルに応じた色分け表示
- **配置**: 独立したLogTabControl内

#### エラーログタブ
- **ヘッダー**: "❌ エラーログ"  
- **機能**: ログ再読み込み、ログクリアボタン
- **表示**: DataGrid（時間、レベル、メッセージ）
- **連携**: error.logファイルと自動同期
- **配置**: 独立したLogTabControl内

#### UI改善の利点
1. **常時表示**: ログが常に見える状態で作業可能
2. **サイズ調整**: 垂直グリッドスプリッターでログ領域のサイズ調整可能
3. **独立性**: データタブのクリアがログに影響しない
4. **アクセシビリティ**: メニューナビゲーション不要

### 4. メッセージボックス置き換え

#### 置き換え完了
- BLGファイル読み込み完了 → `AddOperationLog(LogLevel.Success, ...)`
- カウンターデータ読み込み完了 → `AddOperationLog(LogLevel.Success, ...)`
- パターン適用完了 → `AddOperationLog(LogLevel.Success, ...)`
- CSV出力完了 → `AddOperationLog(LogLevel.Success, ...)`
- タブクリア完了 → `AddOperationLog(LogLevel.Success, ...)`
- パターン設定再読み込み完了 → `AddOperationLog(LogLevel.Success, ...)`

#### エラー・警告の二重出力
エラーや警告メッセージは、ユーザビリティを考慮してメッセージボックスを保持しつつ、ログにも同時出力するよう実装しました。

```csharp
// 例: エラー処理
catch (Exception ex)
{
    AddOperationLog(LogLevel.Error, $"処理に失敗しました: {ex.Message}");
    MessageBox.Show($"処理に失敗しました: {ex.Message}", "エラー", MessageBoxButton.OK, MessageBoxImage.Error);
    LogError($"Detailed error: {ex}");
}
```

### 5. パフォーマンス最適化

- **ログ上限**: 1000件で自動削除
- **UIスレッド対応**: `Dispatcher.CheckAccess()`での安全な更新
- **メモリ効率**: `ObservableCollection`の適切な使用

## 技術的考慮事項

### タブ管理の改善
**初期実装**: 全タブが同一のTabControlで混在
```csharp
// タブクリア時にログタブを保持する複雑な処理
var tabsToRemove = DataTabControl.Items.Cast<TabItem>()
    .Where(tab => tab.Tag?.ToString() != "OperationLog" && tab.Tag?.ToString() != "ErrorLog")
    .ToList();
```

**改善後**: ログタブとデータタブを完全分離
```csharp
// シンプルなデータタブ全削除
DataTabControl.Items.Clear();

// ログタブの初期化
LogTabControl.Items.Add(CreateOperationLogTab());
LogTabControl.Items.Add(CreateErrorLogTab());
```

### ファイルIOの安全性
error.logファイルの読み書きにおいて、ファイルロックやエンコーディングを適切に処理：

```csharp
var lines = File.ReadAllLines(logPath, Encoding.UTF8);
```

### UIの応答性
大量ログ処理時のUI凍結防止のため、非同期処理とDispatcher使用：

```csharp
if (Dispatcher.CheckAccess())
{
    _operationLogs.Insert(0, logEntry);
}
else
{
    Dispatcher.Invoke(() => AddOperationLog(level, message));
}
```
