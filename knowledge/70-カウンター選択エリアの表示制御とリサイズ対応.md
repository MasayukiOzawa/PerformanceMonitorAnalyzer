# カウンター選択エリアの表示制御とリサイズ対応

## 背景
カウンター選択エリアについて、グラフ領域を広く使いたいケースに対応するため、UIとして「表示/非表示切替」「領域サイズ変更」を追加した。

## 実施内容
- `MainWindow.xaml`
  - メイン表示領域の列構成を見直し、カウンター選択エリアとグラフ領域の間に `GridSplitter` を追加。
  - カウンター境界領域にトグルアイコン（◀/▶）を追加し、クリックで表示/非表示を切替可能にした。
  - 右側のスケール設定パネル列を 1 列右に移動してレイアウトを維持。

- `MainWindow.xaml.cs`
  - カウンター選択エリアの状態管理フィールドを追加。
    - `_isCounterPanelVisible`
    - `_lastCounterPanelWidth`
  - 初期化処理 `InitializeCounterPanelControls()` を追加。
  - 表示切替イベント `CounterPanelVisibilityButton_Click` を追加。
  - 内部処理として `HideCounterPanel()`, `ShowCounterPanel()`, `UpdateCounterPanelControls()` を追加。

## 試行と調整ポイント
- 単純な `GroupBox` の `Visibility` 切替だけでは、再表示時の幅が初期化されるため、直前幅を保持する設計に変更。
- 非表示時は `CounterPanelColumn.Width = 0` と `CounterPanelSplitterColumn.Width = 0` を同時に適用し、無駄な余白を残さないようにした。
- 再表示時は `MinWidth` を復元し、`GridSplitter` も再有効化して操作性を維持。

## 検証
- `dotnet build src\PerformanceMonitorAnalyzer\PerformanceMonitorAnalyzer.csproj` でビルド成功（`net10.0-windows`）。
- UI操作として以下を確認。
  - 境界線中央のトグルアイコン（◀/▶）で表示/非表示切替可能
  - 再表示時に直前の横幅が復元される
  - `GridSplitter` で横幅調整が可能

## 追加対応: スケール設定領域の可変幅化
- `MainWindow.xaml`
  - 右側のスケール設定エリア用に `ScalePanelColumn` を追加し、固定幅から可変幅へ変更。
  - グラフ領域との間に `ScalePanelGridSplitter` を追加し、ドラッグで幅調整可能にした。
  - 境界領域に `ScalePanelToggleButton`（◀/▶）を追加し、クリックで表示/非表示を切替可能にした。
- `MainWindow.xaml.cs`
  - スケールパネルの最終幅保持用に `_lastScalePanelWidth` を追加。
  - 手動折りたたみ状態保持用に `_isScalePanelCollapsed` を追加。
  - `UpdateScaleControlVisibility()` で、表示時に前回幅を復元し、非表示時は列幅を 0 にする制御を追加。
  - 非表示時に `MinWidth` 制約が残って幅が十分に縮まらない問題に対し、折りたたみ時は `ScalePanelColumn.MinWidth = 0`、再表示時は `MinWidth = 210` へ戻す制御を追加。

## 追加対応: グラフ操作ボタンと凡例の重なり回避
- 問題: 「ズームをリセット」「グラフをクリップボードにコピー」ボタンが凡例エリア上に重なって表示される場合がある。
- 対応: グラフ操作ボタンをグラフ表示行の重ね描画から独立行へ移動し、凡例エリアと同じ行で重ならないようレイアウトを変更。
- 併せて、統計情報 `GridSplitter` と統計表示領域の行位置を調整して、既存の縦リサイズ機能を維持。
