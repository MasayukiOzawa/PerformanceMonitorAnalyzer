# PerfMon 風カウンター選択 UI

## 概要

左側のカウンター選択 UI を、単一の階層 TreeView から標準パフォーマンスモニターの「カウンターの追加」に近い構成へ変更した。

## 実施内容

- `MainWindow.xaml` のカウンター選択領域を以下の 4 領域に分離した。
  - オブジェクト
  - カウンター
  - インスタンス
  - 追加されたカウンター
- `CounterSelectorItem` を追加し、UI 表示用のカウンター / インスタンス / 追加済みカウンターを扱えるようにした。
- `MainWindow.xaml.cs` で追加済みカウンター一覧を選択状態の正とし、既存の `GetSelectedCounters()` から読み込み、relog コマンド表示、グラフ再描画、パターン適用へ接続した。
- 複数インスタンスを持つオブジェクトでは `<すべてのインスタンス>` を表示し、一括追加できるようにした。
- README と wiki の操作説明を新しい選択 UI に合わせて更新した。
- オブジェクト / カウンター / インスタンス / 追加されたカウンターの各領域を同じ `*` 高さに揃えた。
- `Assets\app-icon.ico`、`ApplicationIcon`、Window `Icon` 設定を復元し、アプリとウィンドウのアイコンが表示されるようにした。
- カウンター / インスタンスの初期表示では行選択のみを行い、チェックは既定で付けないようにした。
- グラフ実装完了を知らせる旧案内表示を削除し、カウンター追加領域の見出しを「カウンターを追加」に変更した。
- グラフ領域への `.blg` ファイルのドラッグ＆ドロップ読み込みを復元した。
- インスタンスなしカウンターで `(総合)` を表示せず、該当オブジェクト選択時はインスタンス欄を非表示にするようにした。
- `MainWindow.xaml.cs` のカウンター選択処理で、実インスタンス / 表示対象インスタンスの抽出とインスタンス欄の表示切替を共通ヘルパーへ整理した。
- 左ペインの視認性を高めるため、オブジェクト / カウンター / インスタンスの選択 UI を `CounterAddDialog` に移動した。
- 左ペインには「カウンターを追加...」ボタン、追加されたカウンター一覧、削除、クリアを残した。
- `CounterSelectionModel` を追加し、ダイアログ側で利用するオブジェクト名、カウンター候補、インスタンス候補、選択カウンターパス生成を分離した。
- ダイアログは標準 PerfMon に近く、「追加」で追加済み一覧へ反映しつつ開いたままにし、「閉じる」で終了する操作にした。
- ダイアログでは、インスタンスを持たないオブジェクトでもインスタンス欄を折りたたまず、空白の一覧として表示するようにした。
- インスタンスを持つオブジェクトでは、カウンター未選択時にインスタンス一覧を無効化し、カウンター選択後にインスタンスを選べるようにした。

## 実装時の注意点

- 既存のデータ読み込みやグラフ描画は `GetSelectedCounters()` を入口にしていたため、この戻り値を追加済みカウンター一覧から生成することで影響範囲を抑えた。
- 既存のパターン適用は TreeView ノードを直接チェックする前提だったため、検索結果の `FullPath` を追加済みカウンター一覧へ追加する形へ変更した。
- 既定のビルド出力先は実行中のアプリでロックされる場合があるため、検証時は一時出力先を指定した。
- モーダル化後も追加済みカウンター一覧を正の状態として維持し、既存の読み込み、relog コマンド表示、グラフ連携、パターン適用への影響を抑えた。
