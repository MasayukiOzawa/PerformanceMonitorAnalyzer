# 28-統計情報表示領域の追加

## 概要
グラフ表示の下に、選択したカウンターの統計情報（平均、最大、最小値）を表示する領域を追加しました。
これにより、ユーザーは選択したデータの統計情報を一目で確認できるようになります。

## 実装詳細

### UI構造の変更 (MainWindow.xaml)

#### 変更前の構造
```xml
<GroupBox Grid.Column="1" Header="グラフ表示">
    <Grid>
        <scottplot:WpfPlot x:Name="PerformanceChart" />
        <!-- データなしメッセージ -->
    </Grid>
</GroupBox>
```

```xml
<GroupBox Grid.Column="1" Header="グラフ表示">
```

### ロジック実装 (MainWindow.xaml.cs)

#### 新規追加メソッド

1. **UpdateStatisticsDisplay()**
   - 統計情報表示の更新を制御
   - グラフにデータがある場合のみ統計エリアを表示
   - 各カウンターの統計情報パネルを作成

2. **CreateStatisticsPanel(string, List<(DateTime, double)>)**
   - 特定カウンターの統計情報パネルを作成
   - スケール設定を考慮した値の計算
   - 平均、最大、最小値の計算とフォーマット
   - 色分けされた統計値の表示

#### 既存メソッドの拡張

1. **UpdateChartVisibility()**
   - 既存のグラフ表示制御に統計情報更新を追加
   ```csharp
   private void UpdateChartVisibility()
   {
       var hasData = _chartSeries.Any();
       NoDataMessagePanel.Visibility = hasData ? Visibility.Collapsed : Visibility.Visible;
       
       // 統計情報表示の更新を追加
       UpdateStatisticsDisplay();
   }
   ```

2. **RefreshCounterInChart()**
   - カウンター更新時に統計情報も更新
   ```csharp
   finally
   {
       _isUpdatingScaleControls = false;
       UpdateScaleControlVisibility();
       
       // 統計情報も更新を追加
       UpdateStatisticsDisplay();
   }
   ```

### 統計計算の詳細

#### データ処理フロー
1. `_counterData`から`PerformanceDataPoint`のリストを取得
2. `(DateTime Timestamp, double Value)`タプルリストに変換
3. スケール設定（`_counterScales`）を適用
4. 統計値（平均、最大、最小）を計算
5. `EstimateUnit()`で単位を推定
6. `FormatValueWithUnit()`でフォーマット
7. UIパネルを作成して表示

#### 統計情報パネルの構成
```
┌─────────────────────────────────────────┐
│ [カウンター名] (省略表示対応)             │
│ 平均: 45.2% | 最大: 98.1% | 最小: 12.3% │
└─────────────────────────────────────────┘
```

- **平均値**: 緑色 (RGB: 0, 120, 0)
- **最大値**: 赤色 (RGB: 180, 0, 0)  
- **最小値**: 青色 (RGB: 0, 0, 180)

### 表示制御

#### 表示条件
- グラフに1つ以上のカウンターが表示されている場合に統計エリアを表示
- カウンターが削除されて空になった場合は統計エリアを非表示

#### 更新タイミング
- カウンター追加時（AddCounterToChart）
- カウンター削除時（RemoveCounterFromChart）
- カウンタースケール変更時（RefreshCounterInChart）
- グラフ初期化時（InitializeChartWithSelectedCounters）

## 技術的特徴

### 既存機能との統合
- 既存の`EstimateUnit()`メソッドによる単位認識
- 既存の`FormatValueWithUnit()`メソッドによる値フォーマット
- 既存の`GetCounterDisplayName()`メソッドによる名前表示
- スケール設定（`_counterScales`）との連携

### パフォーマンス配慮
- 統計計算はLINQのMin/Max/Averageを使用（効率的）
- UIパネルの動的生成（メモリ効率）
- スクロール可能なコンテナ（多数カウンター対応）

### エラーハンドリング
- 統計計算でのエラー時は簡潔なエラーメッセージを表示
- null/空データのチェック
- try-catchによる例外処理

## ユーザーエクスペリエンス

### 利用シーン
1. **システム監視**: CPU、メモリ使用率の統計確認
2. **パフォーマンス分析**: ピーク値と平均値の比較
3. **レポート作成**: 数値データの要約情報取得
4. **問題調査**: 異常値の特定
