# データテーブル領域 UI 改善

## 概要

- 下部データテーブル領域で `TabControl` を使っていたため、多数カウンター読み込み時にタブ見出しが多段化し、表本体の高さを圧迫していました。
- 今回はデータテーブル領域を「一覧 + 詳細」構成へ変更し、左側で表示中カウンターを選び、右側で選択中 1 件の表と統計を確認する UI に整理しました。

## 対応内容

1. `MainWindow.xaml`
   - `DataTabControl` を廃止
   - 左側に表示中カウンター一覧 (`ListBox`)、右側に詳細ビュー (`ContentControl`) を配置
   - `表示中一覧を閉じる` ボタンと件数表示を追加
   - 一覧と詳細の間に横方向の `GridSplitter` を追加
   - データテーブル詳細の `DataGrid` ヘッダーにも共通のソート表示テンプレートを適用し、`▲` / `▼` を表示
2. `MainWindow.xaml.cs`
   - 表示中カウンター一覧を `ObservableCollection<DataTableCounterItem>` で管理
   - 詳細表示更新・一覧同期・空状態表示の専用メソッドを追加
   - 既存の `AddCounterTab` / `RemoveCounterTab` / `ShowDataTablesForCounters` / `RefreshAllDataTabsForCurrentMode` を新 UI 前提へ差し替え
3. `DataTableSelectionState`
   - 一覧の同期と選択維持ロジックを helper として切り出し
   - 選択中カウンター削除時に、前後関係を見て自然に次の項目を選ぶようにしました

## 試行と判断

- 単純な横スクロールタブ化も候補でしたが、表示件数が増えるほど表本体の視認性を確保しにくいため不採用にしました。
- 上部ドロップダウン切替よりも、現在表示中の項目数と対象が見えやすい「一覧 + 詳細」の方が操作感として自然と判断しました。
- WPF の UI 全体を ViewModel 化する大規模変更は避け、一覧同期ロジックだけ helper に切り出してテスト対象を明確化しました。
