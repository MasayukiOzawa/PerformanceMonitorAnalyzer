# 58 - 2並列でのカウンターの読み込み

## 概要
選択されたカウンターを読み込む際に2並列でカウンターを読み込み処理を高速化し、データをプロットする際にはデッドロックが発生しないように直列化したプロット処理を実装しました。

## 実装内容

### 1. BlgFileAnalyzer.cs の変更

#### 追加したメソッド
- `LoadMultipleCounterDataAsync(IEnumerable<string> counterPaths, IProgress<string>? progress = null)`
- `LoadMultipleCounterDataAsync(IEnumerable<string> counterPaths, DateTime? startTime, DateTime? endTime, IProgress<string>? progress = null)`

#### 技術的な実装詳細
```csharp
// 2並列に制限するSemaphore
using var semaphore = new SemaphoreSlim(2, 2);
var results = new ConcurrentBag<CounterInfo>();

// 全てのカウンターを並列で処理
var tasks = counterPathsList.Select(async counterPath =>
{
    await semaphore.WaitAsync();
    try
    {
        var counterInfo = await LoadCounterDataAsync(counterPath, progress);
        results.Add(counterInfo);
        return counterInfo;
    }
    finally
    {
        semaphore.Release();
    }
});

await Task.WhenAll(tasks);
```

#### 追加した using文
```csharp
using System.Collections.Concurrent;
```

### 2. MainWindow.xaml.cs の変更

#### プロット処理の直列化
```csharp
// プロット処理の直列化のためのlockオブジェクト
private readonly object _plotLock = new();
```

#### ExecuteRelogForSelectedCounters メソッドの変更
- 従来の順次処理（foreach）から2並列処理に変更
- プロット処理部分を `lock (_plotLock)` で囲んで直列化

```csharp
// 2並列でカウンターデータを読み込み
List<BlgFileAnalyzer.CounterInfo> counterInfos;
if (useTimeConstraints)

// プロット処理は直列化

        // プロット処理
    }
}
```

## 期待される効果

### パフォーマンス向上
- **読み込み処理**: 2並列実行により最大50%の処理時間短縮が期待される
- **デッドロック回避**: プロット処理の直列化により安定性が向上

### 安全性の向上
- `SemaphoreSlim`による並列数制限でリソース使用量を制御
- `lock`による排他制御でUI操作の競合状態を回避

## 実装のメリット

### 1. 既存コードとの互換性
- 既存の `LoadCounterDataAsync` メソッドはそのまま維持
- 新しいメソッドを追加する形で実装

### 2. エラーハンドリング
- 個別カウンターのエラーは全体の処理を停止させない
- `ConcurrentBag`による安全な結果収集

### 3. 進行状況の報告
- 並列処理中も適切にプログレスを報告
- ユーザーに処理状況を可視化

## 技術的な考慮事項

### 並列処理の制限
- `SemaphoreSlim(2, 2)` で同時実行を2つまでに制限
- リソース枯渇やパフォーマンス劣化を防止

### デッドロック回避
- プロット処理（UI操作）を直列化
- `Dispatcher.InvokeAsync` と `lock` の組み合わせによる安全な実装

### メモリ効率
- `ConcurrentBag` による効率的な結果収集
- 不要なデータのコピーを最小限に抑制
