# 58 - 2並列でのカウンターの読み込み

## 概要
選択されたカウンターを読み込む際に2並列でカウンターを読み込み処理を高速化し、データをプロットする際にはデッドロックが発生しないように直列化したプロット処理を実装しました。

## 実装内容

### 1. BlgFileAnalyzer.cs の変更

#### 追加したメソッド
- `LoadMultipleCounterDataAsync(IEnumerable<string> counterPaths, IProgress<string>? progress = null)`
- `LoadMultipleCounterDataAsync(IEnumerable<string> counterPaths, DateTime? startTime, DateTime? endTime, IProgress<string>? progress = null)`

#### 技術的な実装詳細（第1回実装）
```csharp
// 2並列に制限するSemaphore
using var semaphore = new SemaphoreSlim(2, 2);
var results = new ConcurrentBag<CounterInfo>();

// 全てのカウンターを並列で処理
var tasks = counterPathsList.Select(async counterPath =>
{
    await semaphore.WaitAsync();
    try
    {
        var counterInfo = await LoadCounterDataAsync(counterPath, progress);
        results.Add(counterInfo);
        return counterInfo;
    }
    finally
    {
        semaphore.Release();
    }
});

await Task.WhenAll(tasks);
```

#### 技術的な実装詳細（第2回修正: 順序保持対応）
```csharp
// 2並列に制限するSemaphore
using var semaphore = new SemaphoreSlim(2, 2);
var results = new Dictionary<string, CounterInfo>();
var lockObj = new object();

// 全てのカウンターを並列で処理
var tasks = counterPathsList.Select(async counterPath =>
{
    await semaphore.WaitAsync();
    try
    {
        progress?.Report($"カウンター読み込み開始: {counterPath}");
        var counterInfo = await LoadCounterDataAsync(counterPath, progress);
        
        // 順序を保持するためにDictionaryを使用
        lock (lockObj)
        {
            results[counterPath] = counterInfo;
        }
        
        progress?.Report($"カウンター読み込み完了: {counterPath}");
        return counterInfo;
    }
    finally
    {
        semaphore.Release();
    }
});

await Task.WhenAll(tasks);

// 元の順序を保持してリストに変換
var resultList = counterPathsList.Select(path => results[path]).ToList();
```

## 修正された問題

### 🚨 第1回の問題: CS1996エラー
- **エラー内容**: `lock ステートメントの本体で待機することはできません`
- **原因**: `lock` ステートメント内で `await` を使用していた
- **解決方法**: `lock` を `SemaphoreSlim.WaitAsync()` に変更

### 🚨 第2回の問題: プロット順序の不整合
- **問題内容**: プロットされたグラフが期待したものではない
- **原因**: 
  1. `ConcurrentBag` により要素の順序が保証されない
  2. カウンターごとの個別系列としてのプロットが不完全
- **解決方法**:
  1. `ConcurrentBag` を `Dictionary` + `lock` に変更して順序保持
  2. 結果処理で `for` ループを使用して元の順序を保持
  3. カウンターごとに個別の系列として明確に処理

## 期待される効果

### パフォーマンス向上
- **読み込み処理**: 2並列実行により最大50%の処理時間短縮が期待される
- **デッドロック回避**: プロット処理の直列化により安定性が向上

### 安全性の向上
- `SemaphoreSlim`による並列数制限でリソース使用量を制御
- `SemaphoreSlim`による排他制御でUI操作の競合状態を回避（非同期対応）

### 機能性の向上
- **個別系列**: 一つのカウンターは一つのスレッドで読み込み、一つの系列としてプロット
- **順序保証**: 選択したカウンターの順序でプロットが実行される

## 実装のメリット

### 1. 既存コードとの互換性
- 既存の `LoadCounterDataAsync` メソッドはそのまま維持
- 新しいメソッドを追加する形で実装

### 2. エラーハンドリング
- 個別カウンターのエラーは全体の処理を停止させない
- 安全な結果収集（`ConcurrentBag` → `Dictionary` + `lock`）

### 3. 進行状況の報告
- 並列処理中も適切にプログレスを報告
- ユーザーに処理状況を可視化

## 技術的な考慮事項

### 並列処理の制限
- `SemaphoreSlim(2, 2)` で同時実行を2つまでに制限
- リソース枯渇やパフォーマンス劣化を防止

### デッドロック回避
- プロット処理（UI操作）を直列化
- `Dispatcher.InvokeAsync` と `SemaphoreSlim` の組み合わせによる安全な実装

### メモリ効率
- `Dictionary` による効率的な結果収集と順序保持
- 不要なデータのコピーを最小限に抑制

### 順序保持
- 入力されたカウンターの順序を保持してプロット
- `Dictionary<string, CounterInfo>` + 元リストからの順序復元
