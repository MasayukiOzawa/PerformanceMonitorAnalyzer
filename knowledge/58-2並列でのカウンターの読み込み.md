# 58 - 2並列でのカウンターの読み込み（最終版: 並列処理と安定性の両立）

## 概要
選択されたカウンターを読み込む際に2並列でカウンターを読み込み処理を高速化し、メモリ使用量を制御して大容量ファイルでも安定動作し、データをプロットする際にはデッドロックが発生しないように直列化したプロット処理を実装しました。

## 最終実装内容

### 1. BlgFileAnalyzer.cs の改良

#### 新しい並列処理メソッドの追加

- `LoadMultipleCounterDataParallelAsync(IEnumerable<string> counterPaths, IProgress<string>? progress = null)`
- `LoadMultipleCounterDataParallelAsync(IEnumerable<string> counterPaths, DateTime? startTime, DateTime? endTime, IProgress<string>? progress = null)`

#### 同期版ヘルパーメソッドの追加

- `LoadCounterDataSync(string counterPath)` - スレッドセーフなカウンターデータ読み込み
- `LoadCounterDataWithTimeSync(string counterPath, DateTime? startTime, DateTime? endTime)` - 時間制約付きスレッドセーフ読み込み

#### 技術的特徴（最終版）

```csharp
// 並列処理オプションを設定（最大2並列）
var parallelOptions = new ParallelOptions
{
    MaxDegreeOfParallelism = 2
};

// 結果を順序付きで保存するための配列

await Task.Run(() =>

    Parallel.For(0, counterPathsList.Count, parallelOptions, i =>

        results[i] = counterInfo;

```

## 安全性とパフォーマンスの改善

### 1. メモリ使用量の制御

- 最大サンプル数を50,000に制限（従来の100,000から削減）
- 大容量ファイルでのメモリ使用量を抑制してクラッシュを防止

### 2. エラーハンドリングの強化

- 個別カウンターのエラーが全体処理を停止させない設計
- SQLServerカウンターに対する寛容なエラーハンドリング
- スレッドセーフな進行状況報告

### 3. データ整合性の保証

- タイムスタンプ順のデータソート
- カウンター選択順序の保持（配列インデックス使用）
- 排他制御によるUI更新の安全性確保

## 期待される効果

### ⚡ パフォーマンス向上

- **読み込み処理**: 2並列実行により最大50%の処理時間短縮

## 並列処理 vs 安定性のバランス

この最終実装では以下のバランスを取っています：

1. **並列処理**: カウンター読み込みを2並列で実行（パフォーマンス向上）
2. **安定性**: プロット処理を排他制御で直列化（競合状態回避）
3. **メモリ効率**: サンプル数制限による大容量ファイル対応（クラッシュ回避）
4. **エラー処理**: 個別失敗が全体を停止させない設計（堅牢性向上）

## 学んだ教訓

1. **並列処理の複雑性**: 複雑な並列処理は予期しない問題を引き起こす可能性
2. **メモリ使用量**: 大容量ファイルでは並列処理がメモリ問題を悪化させる場合がある
3. **UI更新の競合**: WPF UIの更新は慎重な排他制御が必要
4. **段階的な実装**: 安定性を確保しながら段階的に機能を追加する重要性
5. **パフォーマンスと安定性**: 最適なバランスポイントを見つけることの重要性
