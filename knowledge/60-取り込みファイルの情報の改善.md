# 60-取り込みファイルの情報の改善

## タスクの概要

Performance Monitor Analyzerアプリケーションにおいて、取り込みファイルの表示情報を改善する作業を実施しました。

## 要件

1. **フルパス表示**: 現在ファイル名のみ表示している箇所をフルパスで表示するように変更
2. **ファイルサイズ表示**: ファイルサイズの項目を新規追加

## 実装内容

### 1. フルパス表示への変更

**変更前**: MainWindow.xaml.cs の710行目
```csharp
FileNameDisplay.Text = $"読み込みファイル: {Path.GetFileName(fileName)}";
```

**変更後**:
```csharp
FileNameDisplay.Text = $"読み込みファイル: {fileName}";
```

### 2. ファイルサイズ表示機能の追加

#### UI コンポーネントの追加
MainWindow.xaml に新しい TextBlock を追加:
```xml
<!-- ファイルサイズ表示 -->
<TextBlock Grid.Row="0" x:Name="FileSizeDisplay" 
         Text="" 
         Margin="5,25,5,5" 
         FontWeight="Bold"
         Foreground="DarkBlue"
         TextWrapping="Wrap"
         Visibility="Collapsed"/>
```

#### ファイルサイズ取得・表示処理
MainWindow.xaml.cs の LoadBlgFileAsync メソッドに追加:
```csharp
// ファイルサイズを表示
try
{
    var fileInfo = new FileInfo(fileName);
    var fileSizeText = FormatFileSize(fileInfo.Length);
    FileSizeDisplay.Text = $"ファイルサイズ: {fileSizeText}";
    FileSizeDisplay.Visibility = Visibility.Visible;
}
catch (Exception ex)
{
    FileSizeDisplay.Text = "ファイルサイズ: 取得できません";
    FileSizeDisplay.Visibility = Visibility.Visible;
    LogError($"ファイルサイズの取得に失敗: {ex.Message}");
}
```

#### ファイルサイズフォーマット用ヘルパーメソッド
```csharp
/// <summary>
/// ファイルサイズを読みやすい形式にフォーマット
/// </summary>
/// <param name="bytes">バイト数</param>
/// <returns>フォーマットされたファイルサイズ文字列</returns>
private string FormatFileSize(long bytes)
{
    if (bytes < 1024)
        return $"{bytes} B";
    
    if (bytes < 1024 * 1024)
        return $"{bytes / 1024.0:F1} KB";
    
    if (bytes < 1024 * 1024 * 1024)
        return $"{bytes / (1024.0 * 1024.0):F1} MB";
    
    return $"{bytes / (1024.0 * 1024.0 * 1024.0):F1} GB";
}
```

### 3. UIレイアウトの調整

既存の表示要素との重複を避けるため、位置調整を実施:

- **ファイルサイズ表示**: Margin="5,25,5,5" (ファイル名の下)
- **コンピューター名表示**: Margin="5,45,5,10" (25px → 45px)
- **データ取得間隔表示**: Margin="5,65,5,10" (45px → 65px)

### 4. UI状態管理の更新

UI状態リセット時にファイルサイズ表示も非表示にするよう処理を追加:
```csharp
FileSizeDisplay.Visibility = Visibility.Collapsed;
```

## 技術的な考慮事項

### エラーハンドリング
- ファイルサイズ取得時のエラーを適切にキャッチし、ユーザーに分かりやすいメッセージを表示
- エラーログにも詳細を記録

### ファイルサイズ表示形式
- B (バイト) / KB (キロバイト) / MB (メガバイト) / GB (ギガバイト) の単位で自動切り替え
- 小数点以下1桁まで表示
- 1024ベースでの計算を使用

### UI/UX の配慮
- 既存のデザインパターンに合わせて統一感のある表示
- 表示順序: ファイル名 → ファイルサイズ → コンピューター名 → データ取得間隔
- フォント、色、スタイルは既存要素と統一

## テスト結果

FormatFileSizeメソッドの単体テストを実施し、以下の形式で正常にフォーマットされることを確認:

- 0 B (0バイト)
- 512 B (512バイト)
- 1.5 KB (1536バイト)
- 1.0 MB (1048576バイト)
- 2.0 GB (2147483648バイト)

## 制約事項

- Windows環境でのみ動作するWPFアプリケーションのため、Linux環境でのビルド・実行テストは実施不可
- 実際のBLGファイルを使用した統合テストは実施していない

## まとめ

要件通りに以下の改善を実装しました:

1. ✅ **フルパス表示**: ファイル名のみからフルパス表示に変更
2. ✅ **ファイルサイズ表示**: 新規機能として追加、適切なフォーマットで表示

最小限の変更で既存機能に影響を与えることなく、要件を満たす実装を完了しました。
