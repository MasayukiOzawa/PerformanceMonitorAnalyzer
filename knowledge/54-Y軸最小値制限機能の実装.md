# Y軸最小値制限機能の実装

## 概要

グラフのY軸の最小値を0に制限し、ユーザーがドラッグ操作を行ってもY軸の最小値が負の値にならないようにする機能を実装しました。

## 実装背景

パフォーマンス監視データでは、CPU使用率、メモリ使用量、ディスクI/O等のメトリクスは基本的に負の値を持ちません。しかし、既存の実装ではユーザーがグラフをドラッグしてズームやパンを行う際に、Y軸の最小値が負の値になってしまう問題がありました。

## 実装内容

### 1. Y軸最小値制限メソッドの追加

```csharp
/// <summary>
/// Y軸の最小値を0に制限する（パフォーマンス監視データは負の値を持たない）
/// </summary>
private void EnsureYAxisMinimumZero()
{
    try
    {
        // 現在のY軸の最小値が0未満の場合は0に設定
        if (PerformanceChart.Plot.Axes.Left.Min < 0)
        {
            PerformanceChart.Plot.Axes.Left.Min = 0;
            System.Diagnostics.Debug.WriteLine("Y軸の最小値を0に制限しました");
        }
    }
    catch (Exception ex)
    {
        System.Diagnostics.Debug.WriteLine($"Y軸最小値制限エラー: {ex.Message}");
    }
}
```

### 2. InitializeChart()メソッドの修正

- Y軸の初期最小値を0に設定
- ユーザーのドラッグ操作後にもY軸の最小値を制限するためのイベントハンドラーを追加

```csharp
// Y軸の最小値を0に制限（パフォーマンス監視データは負の値を持たない）
PerformanceChart.Plot.Axes.Left.Min = 0;

// ユーザーのドラッグ操作後にもY軸の最小値を制限
PerformanceChart.Plot.RenderManager.RenderFinished += (sender, args) =>
{
    EnsureYAxisMinimumZero();
};
```

### 3. AutoScale()呼び出し後の処理追加

以下のメソッドで `AutoScale()` 実行後に `EnsureYAxisMinimumZero()` を呼び出すように修正：

- `AddCounterToChart()` - カウンターをグラフに追加時
- `AddCounterToChartInternal()` - 内部カウンター追加処理時
- `RefreshChartWithCurrentType()` - グラフタイプ変更時

## 技術的詳細

### 使用ライブラリ
- ScottPlot.WPF v5.0.41

### 実装方法
1. **初期設定**: グラフ初期化時にY軸の最小値を0に設定
2. **自動スケール後の制限**: データ追加やグラフ更新時の自動スケール後にY軸最小値を再確認
3. **ユーザー操作後の制限**: ドラッグやズーム操作後のレンダリング完了時にY軸最小値を制限

### ScottPlot v5のAPI使用
- `PerformanceChart.Plot.Axes.Left.Min` - Y軸の最小値を直接設定
- `PerformanceChart.Plot.RenderManager.RenderFinished` - レンダリング完了イベント

## 動作確認

この実装により、以下の動作が確保されます：

1. **初期表示**: グラフが最初に表示される際、Y軸の最小値は0以上になる
2. **データ追加**: 新しいカウンターをグラフに追加してもY軸の最小値は0以上を維持
3. **グラフタイプ変更**: 折れ線グラフと積み重ね面グラフの切り替え時もY軸の最小値は0以上を維持
4. **ユーザー操作**: ドラッグやズーム操作を行ってもY軸の最小値は0未満にならない

## 影響範囲

- **新機能**: Y軸の最小値制限機能が追加された
- **既存機能への影響**: なし（既存のグラフ機能は従来通り動作）
- **パフォーマンス**: 軽微な処理追加のみで、ユーザー体験に影響なし
