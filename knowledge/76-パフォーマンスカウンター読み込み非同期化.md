# パフォーマンスカウンター読み込み非同期化

## 概要

- パフォーマンスカウンター読み込み中に UI スレッドへ頻繁に戻っていた処理を見直し、読み込み中でも画面操作を継続しやすいよう改善した。
- 特に、BLG 初回読み込み時のカウンターツリー構築と、選択カウンター読み込み時の 1 件ごとのグラフ反映が UI 停滞の主因だったため、バックグラウンド処理と UI 反映を分離した。
- あわせて、進捗パネルから読み込みを中止できるようにし、中止時も取得済みデータを可能な範囲で反映するようにした。

## 実施内容

### 1. ツリー構築の分離

- `CounterTreeBuilder` を新設し、カウンターパスの解析・オブジェクト/インスタンス単位のグループ化・`CounterTreeNode` 構築を UI 非依存ロジックへ切り出した。
- `LoadBlgFileAsync()` では `Task.Run(...)` でツリーを事前構築し、UI スレッドでは完成済みノード群を `CounterTreeView.ItemsSource` へ反映するだけに変更した。

### 2. 進捗更新の間引き

- `BufferedProgressReporter` を追加し、バックグラウンド処理からの高頻度 progress 通知を最新メッセージだけ保持する方式へ変更した。
- `DispatcherTimer` で一定間隔ごとに `ProgressStatusText` を更新することで、`Progress<string>` が毎回 UI スレッドへメッセージをポストしていた状態を解消した。
- これにより、従来の `LogError($"Progress: ...")` のような同期 I/O を伴う進捗ログも不要になった。

### 3. 選択カウンター読み込み時の段階反映

- `ExecuteRelogForSelectedCounters()` では、データポイント整形はバックグラウンドで行いつつ、**1 カウンター読み込み完了ごと** に `ApplyLoadedCounterIncrementallyAsync(...)` で UI へ反映する形へ変更した。
- 折れ線グラフでは対象系列だけを追加 / 更新し、積み重ね面グラフではその時点までに読めたカウンターで全体を再描画する。
- データテーブル側は、表示中一覧を維持したまま新規カウンターだけを追加し、既存選択は極力保つようにした。

### 4. 競合操作の制限

- 読み込み中でもウィンドウ操作やログ確認は継続できるようにしつつ、競合しやすいカウンター選択・時間範囲スライダー・再実行ボタンだけを一時的に無効化した。
- さらに、別の BLG 読み込みを重ねて開始しないよう、`LoadBlgFileWithErrorHandlingAsync()` に再入ガードを追加した。

### 5. 読み込み中止と途中結果の反映

- `ProgressGrid` に `読み込みを中止` ボタンを追加し、BLG 初回解析と選択カウンター読み込みの両方で `CancellationTokenSource` による中止要求を受け付けるようにした。
- `BlgFileAnalyzer.LoadCounterDataAsync(...)` は中止時に例外で失敗させるだけではなく、その時点までに取得済みの `CounterDataPoint` を `CounterInfo` に保持して返すよう変更した。
- `GenerateAllCounterPathsAsync(...)` も中止要求を受けた場合は、そこで処理を打ち切りつつ、ここまでに列挙済みのカウンターパスを返すようにした。
- 選択カウンター読み込み側では `CounterLoadDataMapper` を追加し、取得済みデータだけを `PerformanceDataPoint` へ変換して `_counterData` にまとめて反映するようにした。
- 中止時は「処理済み件数」と「描画対象件数」を操作ログへ残し、ユーザーがどこまで反映されたか確認できるようにした。
- さらに、通常完了時も各カウンターの完了ごとに表示が更新されるため、大量カウンター読込時でも途中経過を確認しやすくなった。

## テスト

以下のコマンドでビルド・テストを実施した。

```powershell
dotnet build --nologo -v q -r win-x64 -p:OutputPath=<temp>\app\ -p:SelfContained=false -p:UseAppHost=false src\PerformanceMonitorAnalyzer\PerformanceMonitorAnalyzer.csproj
dotnet build --nologo -v q -p:OutputPath=<temp>\tests\ -p:SelfContained=false -p:UseAppHost=false tests\PerformanceMonitorAnalyzer.Tests\PerformanceMonitorAnalyzer.Tests.csproj
dotnet test --nologo -v q --no-build -p:OutputPath=<temp>\tests\ -p:SelfContained=false -p:UseAppHost=false tests\PerformanceMonitorAnalyzer.Tests\PerformanceMonitorAnalyzer.Tests.csproj
```

## 補足

- 中止時の部分反映は「完了済みカウンターに加え、現在読み込み中だったカウンターの取得済みデータも可能なら描画に使う」方針で実装した。
- 通常の再読み込みでも、旧データを一括で差し替えるのではなく、各カウンターが完了した時点で順次新しい表示へ切り替わる。
- 将来、さらに体感改善が必要になった場合は、`RefreshChartWithCurrentType()` 内の系列構築をバックグラウンドで事前計算する拡張余地がある。
