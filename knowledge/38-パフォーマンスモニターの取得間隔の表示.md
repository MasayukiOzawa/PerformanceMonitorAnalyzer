# 38-パフォーマンスモニターの取得間隔の表示

## タスク概要
PDHを使用してBLGファイルのカウンターのデータ取得間隔を取得し、ファイル名、コンピュータ名を表示している欄に取得間隔を追加する機能を実装しました。

## 実装した機能

### 1. データ取得間隔の計算と表示
- BLGファイルから時間範囲とサンプル数を取得
- 時間範囲 ÷ (サンプル数-1) で取得間隔を計算
- ユーザーフレンドリーな形式で表示（秒、分、時間単位）

### 2. UI要素の追加
- ファイル名、コンピュータ名の下に「取得間隔」を表示
- データが取得できない場合は非表示

## PDH API の活用
- `PdhGetDataSourceTimeRange` APIを使用してBLGファイルのメタデータを取得
- 開始時間、終了時間、サンプル数から取得間隔を算出
- エラーハンドリングにより、データが取得できない場合は適切に処理

## 表示仕様
- **取得間隔の形式**: 
  - 1時間以上: "X.X時間"
  - 1分以上: "X.X分"
  - 1秒以上: "X.X秒"
  - 1ミリ秒以上: "Xミリ秒"
  - データなし: 非表示

## 動作テスト結果
- フォーマット関数のテストが正常に動作することを確認
- 各時間単位での表示が適切であることを検証

## 今後の拡張可能性
1. 取得間隔の詳細情報（最小/最大間隔）の表示
2. 取得間隔に基づく推奨設定の提案
3. 複数BLGファイルの比較時の取得間隔比較機能

## 注意事項
- Windows環境のPDH APIに依存するため、Windows専用機能
- BLGファイルの形式や破損により取得間隔が取得できない場合があります
- 取得間隔が0の場合は非表示として扱われます

## トラブルシューティング
- 取得間隔が表示されない場合: BLGファイルの形式確認、サンプル数の確認
- エラーが発生する場合: PDH APIの権限、BLGファイルの破損チェック
