# 20-ビルド時のシングルバイナリ化

## タスク概要

.NETアプリケーションにおいてビルド時にシングルバイナリとなるように設定を実装し、ユーザビリティ向上のためのバッチファイルを作成しました。

## 実装内容

### 1. プロジェクトファイルの更新

`src/PerformanceMonitorAnalyzer/PerformanceMonitorAnalyzer.csproj`を以下のように変更：

#### 最終的な設定
```xml
<PublishSingleFile>true</PublishSingleFile>
<SelfContained>true</SelfContained>
<IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>
```

### 設定値の意味

- **`PublishSingleFile=true`**: 単一の実行ファイルとして出力
- **`SelfContained=true`**: .NET ランタイムを含む自己完結型パッケージ
- **`IncludeNativeLibrariesForSelfExtract=true`**: ネイティブライブラリの自動展開
- **`PublishTrimmed`を削除**: WPFアプリケーションではサポートされていないため削除

### 2. ビルドコマンド

#### 開発時のビルド
```bash
dotnet build
dotnet run
```

#### シングルバイナリ生成
```bash
# Windows x64
dotnet publish --configuration Release --runtime win-x64 --self-contained true --output publish/win-x64

# Windows x86  
dotnet publish --configuration Release --runtime win-x86 --self-contained true --output publish/win-x86

# Windows ARM64
dotnet publish --configuration Release --runtime win-arm64 --self-contained true --output publish/win-arm64
```

### 3. バッチファイルの作成

ユーザーのリクエストに応じて、2つのバッチファイルを作成しました：

#### `publish.bat` - シングルバイナリ生成用
- 全アーキテクチャ（x64, x86, ARM64）のシングルバイナリを一括生成
- エラーハンドリングと日本語メッセージ
- 出力先ディレクトリの表示

#### `build-and-run.bat` - 開発時用
- プロジェクトのビルドと実行を一括処理
- 引数によるBLGファイル指定に対応
- エラーハンドリング機能

### 4. バッチファイルの特徴

#### `publish.bat`の特徴
```batch
@echo off
cd /d "%~dp0src\PerformanceMonitorAnalyzer"
dotnet publish --configuration Release --runtime win-x64 --self-contained true --output "..\..\publish\win-x64"
# エラーチェック付きで各アーキテクチャを順次ビルド
```

#### `build-and-run.bat`の特徴
```batch
@echo off
cd /d "%~dp0src\PerformanceMonitorAnalyzer"
dotnet build
# 引数チェックとエラーハンドリング付きでアプリケーション実行
```

## 課題と解決策

### 課題1: WPFでのPublishTrimmed非対応
WPFアプリケーションでは`PublishTrimmed=true`がサポートされていない。

**解決策**: プロジェクトファイルから`PublishTrimmed`設定を削除し、ビルドエラーを解消

### 課題2: ユーザビリティの向上
コマンドラインでの複雑なビルドコマンドの実行が困難。

**解決策**: 
- `publish.bat`で全アーキテクチャの一括ビルド
- `build-and-run.bat`で開発時の簡単実行
- 日本語メッセージとエラーハンドリング

### 課題3: RuntimeIdentifierの指定
プロジェクトファイルで固定RIDを指定すると、他のアーキテクチャでのビルドができなくなる。

**解決策**: プロジェクトファイルからRIDを削除し、publishコマンドで動的に指定

## 成果物

1. **自己完結型シングルバイナリ**
   - .NET Frameworkインストール不要
   - 単一の.exeファイルで動作
   - WPF対応の最適化設定

2. **ユーザーフレンドリーなバッチファイル**
   - ワンクリックビルド対応
   - エラーハンドリング付き
   - 日本語メッセージ

3. **マルチアーキテクチャ対応**
   - Windows x64, x86, ARM64
   - 一括ビルド機能

4. **開発環境統合**
   - IDE統合ビルド対応
   - 引数付き実行対応

## 使用方法

### シングルバイナリ生成
```cmd
publish.bat
```

### 開発時の実行
```cmd
build-and-run.bat
```

または引数付きで：
```cmd
build-and-run.bat "C:\Logs\your-file.blg"
```
