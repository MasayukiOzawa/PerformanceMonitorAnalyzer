# 20-ビルド時のシングルバイナリ化

## タスク概要

.NETアプリケーションにおいてビルド時にシングルバイナリとなるように設定を実装しました。

## 実装内容

### 1. プロジェクトファイルの更新

`src/PerformanceMonitorAnalyzer/PerformanceMonitorAnalyzer.csproj`を以下のように変更：

#### 変更前
```xml
<PublishSingleFile>true</PublishSingleFile>
<SelfContained>false</SelfContained>
```

#### 変更後
```xml
<PublishSingleFile>true</PublishSingleFile>
<SelfContained>true</SelfContained>
<PublishTrimmed>true</PublishTrimmed>
<IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>
```

### 設定値の意味

- **`PublishSingleFile=true`**: 単一の実行ファイルとして出力
- **`SelfContained=true`**: .NET ランタイムを含む自己完結型パッケージ
- **`PublishTrimmed=true`**: 未使用アセンブリの除去によるファイルサイズ削減
- **`IncludeNativeLibrariesForSelfExtract=true`**: ネイティブライブラリの自動展開

### 2. ビルドコマンド

#### 開発時のビルド
```bash
dotnet build
dotnet run
```

#### シングルバイナリ生成
```bash
# Windows x64
dotnet publish --configuration Release --runtime win-x64 --self-contained true --output publish/win-x64

# Windows x86  
dotnet publish --configuration Release --runtime win-x86 --self-contained true --output publish/win-x86

# Windows ARM64
dotnet publish --configuration Release --runtime win-arm64 --self-contained true --output publish/win-arm64
```

## 課題と解決策

### 課題1: ローカル開発環境での統合
WPFアプリケーションをIDEから直接ビルドする際にシングルバイナリ設定が有効になるように設定。

**解決策**: プロジェクトファイルに適切なプロパティを設定することで、ローカルビルド時も自動的にシングルバイナリ化

### 課題2: RuntimeIdentifierの指定
プロジェクトファイルで固定RIDを指定すると、他のアーキテクチャでのビルドができなくなる。

**解決策**: プロジェクトファイルからRIDを削除し、publishコマンドで動的に指定

## 成果物

1. **自己完結型シングルバイナリ**
   - .NET Frameworkインストール不要
   - 単一の.exeファイルで動作
   - ファイルサイズ最適化済み

2. **ローカル開発環境での簡単ビルド**
   - IDE統合ビルド対応
   - 複数アーキテクチャ対応

3. **マルチアーキテクチャ対応**
   - Windows x64, x86, ARM64
