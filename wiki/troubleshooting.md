# トラブルシューティングガイド

## 一般的な問題と解決方法

### 🔧 BLGファイル読み込みエラー

#### 1. SQLServerカウンターでのエラー

**症状:**
```
カウンター '\\Server\SQLServer:Buffer Manager\Buffer cache hit ratio' の読み込みに失敗: Index was out of range
```

**原因:**
- SQLServerサービスが実行されていない
- BLGファイルにSQLServerカウンターが記録されていない
- カウンターパスの形式が不正

**解決方法:**
1. SQLServerサービスの状態を確認
2. アプリケーションは警告として処理し、他のカウンターの読み込みは継続されます
3. エラーログで詳細な原因を確認

**改善された処理（v2.6以降）:**
- SQLServerカウンターのエラーは警告として表示
- 全体の処理は停止せず、他のカウンターは正常に読み込まれます

#### 2. PDH APIエラー

**症状:**
```
PDH error: No object (0xC0000BB8)
```

**原因と対処:**

| エラーコード | 意味 | 対処方法 |
|-------------|------|----------|
| `PDH_CSTATUS_NO_OBJECT` | オブジェクトが見つからない | カウンターのオブジェクト名を確認 |
| `PDH_CSTATUS_NO_INSTANCE` | インスタンスが見つからない | インスタンス名の存在を確認 |
| `PDH_CSTATUS_NO_COUNTER` | カウンターが見つからない | カウンター名のスペルを確認 |
| `PDH_ENTRY_NOT_IN_LOG_FILE` | BLGファイルに記録されていない | 別の期間のBLGファイルを使用 |

### 📊 グラフ表示の問題

#### 1. グラフが表示されない

**原因:**
- データポイントがない
- 時間範囲が不適切
- カウンター選択が未完了

**解決方法:**
1. カウンターを選択し、「実行」ボタンをクリック
2. 時間範囲スライダーで適切な範囲を設定
3. ログでデータ読み込み状況を確認

#### 2. パフォーマンスが低下する

**原因:**
- 大量のデータポイント
- 多数のカウンター同時表示

**解決方法:**
1. 時間範囲を狭める
2. 表示するカウンター数を減らす
3. データポイントのサンプリング間隔を調整

### 🏗️ ビルドとデプロイメントの問題

#### 1. Windows以外でのビルドエラー

**症状:**
```
error MSB4019: The imported project "Microsoft.NET.Sdk.WindowsDesktop" was not found
```

**原因:**
WPF機能はWindows専用のため、他のOS環境ではビルドできません。

**解決方法:**
1. Windows環境でビルドを実行
2. GitHub Actionsなどのクラウドビルドを利用
3. シングルバイナリ形式での配布を検討

#### 2. シングルバイナリ実行時のエラー

**症状:**
アプリケーション起動時に設定ファイルが見つからない

**解決方法:**
1. 設定ファイル（config/）を実行ファイルと同じディレクトリに配置
2. プロジェクト設定で`CopyToOutputDirectory="PreserveNewest"`を確認

### 🎯 カウンターパターンの問題

#### 1. パターンが適用されない

**原因:**
- YAML設定ファイルの構文エラー
- ファイルパスの問題

**解決方法:**
1. `config/counter-patterns.yaml`の構文を確認
2. エラーログでYAML解析エラーを確認
3. サンプルパターンと比較して修正

#### 2. 一部のカウンターが見つからない

**症状:**
```
パターン適用時に一部のカウンターが未検出
```

**解決方法:**
1. BLGファイルに目的のカウンターが記録されているか確認
2. カウンターパスの表記を確認（大文字小文字、スペース等）
3. ワイルドカードパターンの使用を検討

### 🔍 デバッグとログ

#### ログの確認方法

1. **アプリケーション内ログ**
   - メイン画面下部のログ表示エリア
   - 詳細なエラーメッセージと処理状況

2. **プログレスメッセージ**
   - 各処理段階での進行状況
   - エラー発生時の詳細情報

3. **デバッグ出力**
   - Visual Studioのデバッグウィンドウ
   - コンソール出力（Debug.WriteLine）

#### 問題報告時の情報

問題を報告する際は、以下の情報を含めてください：

1. **環境情報**
   - OS バージョン
   - .NET ランタイムバージョン
   - アプリケーションバージョン

2. **エラー情報**
   - 正確なエラーメッセージ
   - エラー発生時の操作手順
   - スクリーンショット

3. **ファイル情報**
   - BLGファイルのサイズと作成日時
   - 使用したカウンターパターン
   - 設定ファイルの内容

## パフォーマンス最適化

### 1. 大容量BLGファイルの処理

**推奨設定:**
- 時間範囲を必要最小限に設定
- 同時選択カウンター数を50個以下に制限
- PDH API方式を使用（relog.exe方式より高速）

### 2. メモリ使用量の最適化

**対策:**
- 不要なカウンターの選択を解除
- 処理完了後のデータクリア
- 大量データ処理時のバッチ処理

### 3. 応答性の向上

**設定:**
- 非同期処理の活用
- プログレス表示の更新頻度調整
- UIスレッドブロッキングの回避

## 既知の制限事項

### 1. プラットフォーム制限
- BLG解析機能はWindows環境専用
- PDH APIはWindowsネイティブライブラリ依存

### 2. データ制限
- 単一BLGファイルのみサポート
- リアルタイム監視は非対応

### 3. 表示制限
- 同時表示カウンター数の実用的上限：100個
- グラフのズーム・パン機能は基本レベル
