# 使用方法ガイド

## Performance Monitor Analyzer の基本的な使い方

### WPF アプリケーションの使用方法（Windows専用）

#### 1. アプリケーション起動
```bash
cd src/PerformanceMonitorAnalyzer
dotnet build
dotnet run
```

#### 2. BLGファイルの読み込み
**方法1: PDH API方式（従来）**
1. メニューバーから「ファイル」→「BLGファイルを開く」を選択
2. ファイルダイアログでBLGファイルを選択
3. カウンター一覧が左側パネルに表示される

**方法2: relog.exe方式（推奨）**
1. メニューバーから「ファイル」→「BLGファイルを開く（relog.exe使用）」を選択
2. ファイルダイアログでBLGファイルを選択
3. カウンター一覧と**時間範囲選択パネル**が表示される
4. **relog実行状況パネル**で処理の進捗確認が可能

#### 📋 ファイル情報表示
BLGファイル読み込み後、左側パネル上部に以下の情報が表示されます：

**🗂️ ファイル情報**
- **読み込みファイル**: 選択したBLGファイルのフルパス
- **ファイルサイズ**: ファイルサイズ（B/KB/MB/GB形式で自動表示）
- **コンピューター**: データ取得元のコンピューター名
- **取得間隔**: パフォーマンスデータのサンプリング間隔

例：
```
読み込みファイル: D:\Git\PerformanceMonitorAnalyzer\sample\DataCollector01.blg
ファイルサイズ: 15.2 MB
コンピューター: NUC-102
取得間隔: 5.0秒
```

#### 🔥 3. 新機能: 時間範囲選択（relog.exe方式のみ）
1. **自動時間範囲検出**: BLGファイル読み込み時に全期間が自動表示
2. **スライダー操作**: 
   - 上のスライダー：開始時刻を調整
   - 下のスライダー：終了時刻を調整
3. **リアルタイム表示**: 選択範囲の時刻と期間が即座に更新
4. **再読み込み**: 「🔄 この時間範囲で再読み込み」ボタンで指定範囲のみ解析

#### 🚀 4. 新機能: relog実行制御
1. **実行状況確認**: 左パネルの「🔄 relog実行状況」で処理中カウンターと進捗を確認
2. **チェックボックス選択**: カウンターをチェックすると**自動的にデータ読み込み開始**
3. **手動実行**: 
   - 複数カウンターをチェック後
   - 「🚀 選択されたカウンターでrelog実行」ボタンで一括処理
   - 成功・エラー件数の詳細レポート表示

#### 5. カウンターの選択とデータ表示

**⭐ 新機能: 階層チェックボックス**

Performance Monitor Analyzerでは、パフォーマンスカウンターの選択を3つのレベルで行うことができます：

**📁 オブジェクトレベル（一括選択）**
- オブジェクト名（例：`Processor`、`Memory`など）のチェックボックスをクリック
- そのオブジェクト配下の**全インスタンス・全カウンター**が自動で選択される
- 大量のカウンターを一度に選択したい場合に便利

**📂 インスタンスレベル（中粒度選択）** 
- インスタンス名（例：`_Total`、`0`、`1`など）のチェックボックスをクリック
- そのインスタンス配下の**全カウンター**が自動で選択される
- 特定のCPUコアやプロセスのみを監視したい場合に便利

**📄 カウンターレベル（個別選択）**
- 個別のカウンター名（例：`% Processor Time`）のチェックボックスをクリック
- そのカウンターのみが選択される
- 特定の指標のみを監視したい場合に便利

**三状態チェックボックス表示**
- ✅ **全選択**: 配下の全要素が選択済み
- ❌ **全解除**: 配下の全要素が未選択  
- ⚠️ **部分選択**: 一部の要素のみ選択済み（親レベルで表示）

**選択の基本操作**
1. 左側のカウンター一覧から表示したいカウンターにチェック
2. **自動データ読み込み**: relog.exe方式では、チェック時に自動でCSVデータ読み込み開始
3. 選択したカウンターが**タブ形式のデータテーブル**とグラフに表示される
4. 複数のカウンターを同時に選択可能（各カウンターが個別タブで表示）

**🧭 カウンター選択エリアの表示制御**
- 選択エリアとグラフの境界線中央にある **トグルアイコン（◀/▶）** で表示/非表示を切り替え
- 選択エリアとグラフの境界線はドラッグ調整せず、トグル操作のみを提供
- メニューバーの **表示** → **すべて折りたたむ / すべて展開する**、または **グラフ表示上段のボタン** で、カウンター / 凡例 / 統計情報 / スケール設定 / 下部データテーブル・ログ領域を一括制御可能

**注意**: グラフ機能が実装されました！選択したカウンターが右側のグラフエリアに時系列グラフとして表示されます。

#### 6. グラフ表示（新機能）
- **📊 グラフタイプ切り替え**: 折れ線グラフと積み重ね面グラフを選択可能
  - **折れ線グラフ（デフォルト）**: 個別カウンターの変化を確認
  - **積み重ね面グラフ**: 複数カウンターの合計や構成比を確認
- **📉 値モード切り替え**: グラフタイプの右側で `Raw / 差分` を切り替え可能
- **📊 自動グラフ化**: チェックボックス選択されたカウンターが右側エリアに時系列グラフとして自動表示
- **複数カウンター対応**: 異なる色の線で複数カウンターを同時表示
- **リアルタイム連携**: チェックボックスの状態変更に応じてグラフも即座に更新
- **高性能描画**: ScottPlot.WPFによる高速で美しいグラフ描画
- **🆕 統計情報表示**: グラフ下部に選択したカウンターの統計情報（平均・最大・最小値）を自動表示
- **🔥 改善された凡例表示**: 大量インスタンス時でもグラフを覆わない独立凡例機能
  - **📋 凡例の表示/非表示**: 「📋 凡例を表示」ボタンで凡例パネルの表示切り替え
  - **↔ 境界トグル**: グラフと凡例の境界線中央のトグルボタンからも、凡例パネルをワンクリックで折りたたみ/再表示可能
  - **固定境界**: 凡例の境界線はドラッグ調整せず、トグル操作のみを提供
  - **スクロール可能**: 大量のシリーズでもスクロール表示で全て確認可能
  - **個別制御**: 各シリーズのチェックボックスで表示/非表示を個別制御
  - **折れ線ハイライト**: 凡例の `☆` ボタンで複数系列を太線（強調）表示に切り替え可能（折れ線グラフ時のみ）
  - **色見本表示**: 各シリーズの色を視覚的に確認可能
  - **現在値表示**: 各シリーズの最新データポイント値をリアルタイム表示
  - **一括操作**: 「すべて表示」「すべて非表示」ボタンで全シリーズのチェック状態を一括制御（凡例や関連パネルの表示状態は維持）
- **スケール適用方針**: グラフ描画時の倍率は各カウンターの手動スケール（`_counterScales`）のみを適用し、自動正規化は行いません
- **スケール設定パネルの表示制御**: グラフ右側の境界中央にあるトグルアイコンで、スケール設定パネルを表示/非表示できます

**📊 グラフタイプの使い分け**
- **折れ線グラフ**: 
  - 個別カウンターの時系列変化を確認したい場合
  - CPU使用率、メモリ使用量などの独立した指標の監視
- **積み重ね面グラフ**: 
  - 複数カウンターの合計値や構成比を確認したい場合
  - プロセス別CPU使用率、ディスク別容量使用量などの分析
  - 全体に対する各要素の寄与度を視覚的に把握
  - 面の境界線は細めのアウトラインで描画し、系列が多い場合でも重なりを追いやすく改善

**🆕 統計情報表示機能**
- **📋 クリップボードコピー**: グラフを画像としてクリップボードにコピー可能

**📋 新機能: グラフのクリップボードコピー**

グラフを画像としてクリップボードにコピーして、他のアプリケーションに貼り付けることができます。

**コピー方法（3つの方法）**
1. **ボタン**: グラフエリア下部の「📋 グラフをクリップボードにコピー」ボタンをクリック
2. **メニューバー**: 「グラフ」→「グラフをクリップボードにコピー」を選択
3. **キーボードショートカット**: `Ctrl+C` キーを押す
4. **右クリックメニュー**: グラフエリアで右クリック→「グラフをクリップボードにコピー」を選択
   - 右クリックはコンテキストメニュー専用で、右ドラッグによるズームは無効です

**Y軸範囲の手動設定**
- グラフ表示上部の左側にある `Y軸` 入力欄で **下限** と **上限** を指定し、`適用` を押すとY軸範囲を固定できます。
- `自動` を押すと手動設定を解除し、データに応じた自動範囲設定に戻ります。
- 上限は下限より大きい値を指定してください。
- Y軸の目盛りと軸線は表示し、Y軸ラベルは表示しません。必要に応じて、上部の `Y軸` 入力欄から下限 / 上限を調整してください。

**使用条件**
- グラフにデータが表示されている場合のみ利用可能
- カウンターが選択されていない場合は、コピーボタンとメニューが無効化されます

**活用例**
- PowerPointプレゼンテーションへの貼り付け
- Wordドキュメントでのレポート作成
- 画像編集ソフトでの加工
- チームでのデータ共有

**⭐ 統計情報表示機能**
- **自動計算**: 選択されたカウンターの平均値、最大値、最小値を自動計算
- **フォーマット表示**: 単位付きの読みやすい形式で表示（%、MB、/sec等）
- **色分け表示**: 平均値（緑）、最大値（赤）、最小値（青）で色分け
- **スケール対応**: グラフのスケール設定を統計計算にも反映
- **境界操作**: グラフ下部との境界線をドラッグして高さを変更でき、中央の `▲/▼` トグルで統計情報エリアを最小化/再表示できます
- **表示例**:
  ```

  平均: 23.4% | 最大: 87.2% | 最小: 8.1%

  ```

#### 7. データテーブルの確認
- 画面下部の**タブコントロール**で各カウンターの詳細データを確認
- **カウンター別タブ**: 選択した各カウンターが独立したタブで表示
- **統計情報表示**: データ数、平均、最大、最小、標準偏差、取得期間
- **時間、値、フォーマット済み値、単位、カウンター名**が表形式で表示
- **CSV出力**: 各タブで「CSV出力」ボタンクリックで個別ファイル出力可能

#### 8. データテーブル / ログ領域のサイズ調整
- **動的リサイズ**: グラフ表示エリアとデータテーブルの境界にあるグレーの分割線をドラッグして、データテーブル領域の高さを自由に調整できます
- **トグル操作**: 境界線中央の `▲/▼` トグルで、下部のデータテーブル / ログ表示エリア全体を最小化 / 再表示できます
- **操作方法**: 境界線にマウスカーソルを合わせると矢印カーソルに変わり、ドラッグでサイズを変更
- **制約**: 最小100ピクセル、最大750ピクセルの範囲で調整可能
- **用途**: 大量のデータを表示する際や、グラフ表示により多くのスペースを確保したい場合に便利

#### 9. 便利機能
- **すべて解除**: 全オブジェクトのチェックボックスを一度に解除（全カウンター解除）
- **すべてのタブを閉じる**: 開いているデータテーブルタブを一括クリア
- **全データをCSV出力**: 全カウンターのデータを統合した1つのCSVファイルに出力
- **🚀 選択されたカウンターを読み込み**: チェック済みのカウンターを一括でrelog処理
  - 読み込み完了後、一括スケールは自動で `1.0` に戻され、表示中カウンターへ適用されます
  - ボタン実行前でも、チェック状態と時間範囲に応じた `relog.exe コマンド` が表示されます

## 🎯 BLG解析方法の選択指針

### PDH API方式を選ぶ場合
- 小さなBLGファイル（数百MB以下）
- 全期間のデータが必要
- Windows環境の互換性を重視

### relog.exe方式を選ぶ場合（推奨）
- 大容量BLGファイル（数GB以上）
- 特定時間範囲のみ解析したい
- より安定した動作を求める
- 詳細な進捗確認が必要

## 出力ファイル

### JSONデータファイル
- **ファイル名**: `sample_performance_data.json`（サンプルデータの場合）
- **内容**: 全カウンターのタイムスタンプ付きデータ
- **形式**: JSON形式で構造化されたデータ

### エラーログ
- **ファイル名**: `error.log`
- **内容**: アプリケーションエラーと診断情報
- **場所**: 実行ディレクトリ

| カウンター名 | 説明 | 単位 |

## ⭐新機能: 時間範囲選択機能

### 概要
relog.exe方式でBLGファイルを開く際に、特定の時間範囲のみを選択してデータ解析できる機能です。大容量BLGファイルの効率的な解析が可能になります。

### 使用方法

#### 1. BLGファイルを開く
- **ファイル > BLGファイルを開く（relog.exe使用）** を選択
- または **ファイル > サンプルBLGファイルを読み込み（relog.exe）** を選択

#### 2. 時間範囲の確認
ファイルの読み込みが完了すると、左側のカウンター一覧の上部に「📅 時間範囲選択」グループが表示されます。

#### 3. 時間範囲の選択
- **上段スライダー**: 開始時刻を調整
- **下段スライダー**: 終了時刻を調整
- スライダーを動かすと、選択中の時間範囲がリアルタイムで表示されます

#### 4. 再読み込み実行
- 「🔄 この時間範囲で再読み込み」ボタンをクリック
- 選択された時間範囲のデータのみでBLGファイルが再解析されます

### 機能詳細

#### 表示情報
- **全期間**: BLGファイル全体の時間範囲と総期間
- **選択範囲**: 現在選択中の時間範囲と期間

#### スライダー操作
- 0% = 開始時刻、100% = 終了時刻として操作
- 開始時刻が終了時刻を超えないよう自動調整
- 終了時刻が開始時刻より前にならないよう自動調整

#### パフォーマンス効果
- **処理速度**: 必要な時間範囲のみ処理するため大幅高速化
- **メモリ使用量**: 時間範囲に比例してメモリ使用量削減
- **ディスク容量**: 一時CSVファイルサイズ削減

### 使用例

#### ケース1: 24時間BLGファイルから障害発生時間のみ抽出
1. 全期間: 2024/01/15 00:00:00 ～ 2024/01/16 00:00:00 (24.0時間)
2. 障害時間: 13:00～15:00に問題が発生
3. スライダーで該当時間を選択: 54% ～ 62%
4. 選択範囲: 2024/01/15 13:00:00 ～ 2024/01/15 15:00:00 (2.0時間)
5. 再読み込み実行 → 2時間分のデータのみで高速解析

#### ケース2: 特定時間帯のピーク分析
1. 業務時間外の解析を除外
2. 9:00～18:00の範囲を選択
3. 業務時間のパフォーマンス特性を集中分析

### 技術的詳細
- relog.exe の `-b` (begin) と `-e` (end) パラメータを使用
- 時刻フォーマット: `MM/dd/yyyy-HH:mm:ss`
- relog.exe の `-q` オプションでBLGファイル情報を事前取得

## ヒントとコツ

### パフォーマンス最適化
- 大きなBLGファイルを扱う場合は、まず時間範囲を絞り込んでから必要なカウンターを選択
- 時間範囲選択により処理時間とメモリ使用量を大幅削減可能
- グラフの表示範囲を制限することで応答性が向上

### データ分析
- JSONファイルを外部ツール（Excel、Python等）で更に詳細分析可能
- 統計情報を参考にシステムのボトルネックを特定

### トラブルシューティング
- エラーが発生した場合は `error.log` を確認
- BLGファイルが破損している場合は別のファイルで試行
- メモリ不足の場合は小さなBLGファイルから始める

## 🖥️ ウィンドウサイズ表示・設定機能

### 概要
アプリケーション画面の右下のステータスバーに現在のウィンドウサイズが表示され、クリック操作で手動でウィンドウサイズを設定できる機能です。

### 基本機能

#### ステータスバー表示
- **場所**: アプリケーション画面の最下部（ステータスバー右側）
- **表示内容**: `ウィンドウサイズ: 1200×800` 形式
- **リアルタイム更新**: ウィンドウサイズ変更時に自動更新
- **ウィンドウ状態表示**: 最大化/最小化時は状態も表示（例：`ウィンドウサイズ: 1920×1080 (最大化)`）

#### 手動サイズ設定
1. ステータスバーのウィンドウサイズ表示をクリック
2. **ウィンドウサイズ設定ダイアログ** が開く
3. 希望のサイズまたは状態を設定
4. **適用** ボタンでウィンドウサイズを変更

### 設定ダイアログの使用方法

#### 現在の設定確認
- **サイズ**: 現在のウィンドウの幅×高さ
- **状態**: 現在のウィンドウ状態（通常/最大化/最小化）

#### ウィンドウ状態選択
- **🔳 通常**: 指定したサイズでウィンドウを表示
- **🔲 最大化**: 画面全体にウィンドウを最大化
- **🗕 最小化**: ウィンドウをタスクバーに最小化

#### サイズ指定（通常状態の場合）
- **幅・高さ**: テキストボックスで数値入力
- **有効範囲**: 幅 400-3840、高さ 300-2160
- **プリセットボタン**: 一般的な解像度をワンクリック設定
  - `1024×768` - XGA解像度
  - `1280×720` - HD解像度
  - `1366×768` - WXGA解像度  
  - `1920×1080` - Full HD解像度

#### プレビュー機能
- 設定変更時にリアルタイムで新しいサイズをプレビュー表示
- 「新しいサイズ: 1920×1080 (通常)」形式で表示

### 使用例

#### ケース1: 特定の解像度に設定
1. ステータスバーのウィンドウサイズ表示をクリック
2. **通常** ラジオボタンを選択
3. プリセットボタン `1920×1080` をクリック
4. プレビューで「新しいサイズ: 1920×1080 (通常)」を確認
5. **適用** ボタンをクリック

#### ケース2: カスタムサイズに設定
1. ステータスバーのウィンドウサイズ表示をクリック
2. **通常** ラジオボタンを選択
3. 幅テキストボックスに `1600`、高さテキストボックスに `900` を入力
4. プレビューで設定内容を確認
5. **適用** ボタンをクリック

#### ケース3: 最大化に切り替え
1. ステータスバーのウィンドウサイズ表示をクリック
2. **🔲 最大化** ラジオボタンを選択
3. プレビューで「新しいサイズ: 最大化」を確認
4. **適用** ボタンをクリック

### 注意事項

#### 制限事項
- 最大化/最小化を選択した場合、サイズ指定は無視されます
- 設定したサイズが画面サイズを超える場合、システムが自動調整します
- 無効な値が入力された場合、現在の値が維持されます

#### エラーハンドリング
- 数値以外の文字が入力された場合は警告メッセージを表示
- 範囲外の値が入力された場合は有効範囲を表示
- 設定適用中にエラーが発生した場合はエラーダイアログを表示

## 制限事項

- BLGファイルの実際の解析はWindows環境でのみ対応
- 非常に大きなBLGファイル（数GB以上）の場合、メモリ使用量に注意
- リアルタイム監視機能は現在未実装
