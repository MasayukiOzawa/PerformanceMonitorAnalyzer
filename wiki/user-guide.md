# 使用方法ガイド

## Performance Monitor Analyzer の基本的な使い方

### WPF アプリケーションの使用方法（Windows専用）

#### 1. アプリケーション起動
```bash
cd src/PerformanceMonitorAnalyzer
dotnet build
dotnet run
```

#### 2. BLGファイルの読み込み
1. メニューバーから「ファイル」→「BLGファイルを開く」を選択
2. ファイルダイアログでBLGファイルを選択
3. カウンター一覧が左側パネルに表示される

#### 3. カウンターの選択とデータ表示
1. 左側のカウンター一覧から表示したいカウンターにチェック
2. 選択したカウンターがデータテーブルに表示される
3. 複数のカウンターを同時に選択可能

**注意**: グラフ機能は現在開発中です。データテーブルでの確認が可能です。

#### 4. データテーブルの確認
- 画面下部のタブで各カウンターの詳細データを確認
- 時間、値、カウンター名が表形式で表示

#### 5. データテーブル領域のサイズ調整（新機能）
- **動的リサイズ**: グラフ表示エリアとデータテーブルの境界にあるグレーの分割線をドラッグして、データテーブル領域の高さを自由に調整できます
- **操作方法**: 境界線にマウスカーソルを合わせると矢印カーソルに変わり、ドラッグでサイズを変更
- **制約**: 最小100ピクセル、最大600ピクセルの範囲で調整可能
- **用途**: 大量のデータを表示する際や、グラフ表示により多くのスペースを確保したい場合に便利

#### 6. 便利機能
- **すべて選択**: 全カウンターを一度に選択
- **すべて解除**: 全カウンターの選択を解除

## 出力ファイル

### JSONデータファイル
- **ファイル名**: `sample_performance_data.json`（サンプルデータの場合）
- **内容**: 全カウンターのタイムスタンプ付きデータ
- **形式**: JSON形式で構造化されたデータ

### エラーログ
- **ファイル名**: `error.log`
- **内容**: アプリケーションエラーと診断情報
- **場所**: 実行ディレクトリ

## サポートするカウンター例

| カウンター名 | 説明 | 単位 |
|-------------|------|------|
| `\Processor(_Total)\% Processor Time` | CPU使用率 | パーセント |
| `\Memory\Available MBytes` | 利用可能メモリ | メガバイト |
| `\PhysicalDisk(_Total)\Disk Reads/sec` | ディスク読み取り速度 | 回/秒 |
| `\PhysicalDisk(_Total)\Disk Writes/sec` | ディスク書き込み速度 | 回/秒 |

| `\Process(_Total)\Working Set` | プロセス作業セット | バイト |

## ⭐新機能: 時間範囲選択機能

### 概要
relog.exe方式でBLGファイルを開く際に、特定の時間範囲のみを選択してデータ解析できる機能です。大容量BLGファイルの効率的な解析が可能になります。

### 使用方法

#### 1. BLGファイルを開く
- **ファイル > BLGファイルを開く（relog.exe使用）** を選択
- または **ファイル > サンプルBLGファイルを読み込み（relog.exe）** を選択

#### 2. 時間範囲の確認
ファイルの読み込みが完了すると、左側のカウンター一覧の上部に「📅 時間範囲選択」グループが表示されます。

#### 3. 時間範囲の選択
- **上段スライダー**: 開始時刻を調整
- **下段スライダー**: 終了時刻を調整
- スライダーを動かすと、選択中の時間範囲がリアルタイムで表示されます

#### 4. 再読み込み実行
- 「🔄 この時間範囲で再読み込み」ボタンをクリック
- 選択された時間範囲のデータのみでBLGファイルが再解析されます

### 機能詳細

#### 表示情報
- **全期間**: BLGファイル全体の時間範囲と総期間
- **選択範囲**: 現在選択中の時間範囲と期間

#### スライダー操作
- 0% = 開始時刻、100% = 終了時刻として操作
- 開始時刻が終了時刻を超えないよう自動調整
- 終了時刻が開始時刻より前にならないよう自動調整

#### パフォーマンス効果
- **処理速度**: 必要な時間範囲のみ処理するため大幅高速化
- **メモリ使用量**: 時間範囲に比例してメモリ使用量削減
- **ディスク容量**: 一時CSVファイルサイズ削減

### 使用例

#### ケース1: 24時間BLGファイルから障害発生時間のみ抽出
1. 全期間: 2024/01/15 00:00:00 ～ 2024/01/16 00:00:00 (24.0時間)
2. 障害時間: 13:00～15:00に問題が発生
3. スライダーで該当時間を選択: 54% ～ 62%
4. 選択範囲: 2024/01/15 13:00:00 ～ 2024/01/15 15:00:00 (2.0時間)
5. 再読み込み実行 → 2時間分のデータのみで高速解析

#### ケース2: 特定時間帯のピーク分析
1. 業務時間外の解析を除外
2. 9:00～18:00の範囲を選択
3. 業務時間のパフォーマンス特性を集中分析

### 技術的詳細
- relog.exe の `-b` (begin) と `-e` (end) パラメータを使用
- 時刻フォーマット: `MM/dd/yyyy-HH:mm:ss`
- relog.exe の `-q` オプションでBLGファイル情報を事前取得

## ヒントとコツ

### パフォーマンス最適化
- 大きなBLGファイルを扱う場合は、まず時間範囲を絞り込んでから必要なカウンターを選択
- 時間範囲選択により処理時間とメモリ使用量を大幅削減可能
- グラフの表示範囲を制限することで応答性が向上

### データ分析
- JSONファイルを外部ツール（Excel、Python等）で更に詳細分析可能
- 統計情報を参考にシステムのボトルネックを特定

### トラブルシューティング
- エラーが発生した場合は `error.log` を確認
- BLGファイルが破損している場合は別のファイルで試行
- メモリ不足の場合は小さなBLGファイルから始める

## 制限事項

- BLGファイルの実際の解析はWindows環境でのみ対応
- 非常に大きなBLGファイル（数GB以上）の場合、メモリ使用量に注意
- リアルタイム監視機能は現在未実装
