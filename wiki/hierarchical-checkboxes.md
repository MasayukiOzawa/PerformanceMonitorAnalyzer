# 階層チェックボックス機能

## 概要

Performance Monitor Analyzerの階層チェックボックス機能により、パフォーマンスカウンターの選択を3つの粒度で効率的に行うことができます。

## 機能詳細

### 階層構造

```
☑️ Processor                          (オブジェクトレベル)
  ☑️ _Total                           (インスタンスレベル)
    ☑️ % Processor Time               (カウンターレベル)
    ☑️ % User Time                    (カウンターレベル)
    ☑️ % Privileged Time              (カウンターレベル)
  ☑️ 0                                (インスタンスレベル)
    ☑️ % Processor Time               (カウンターレベル)
    ☑️ % User Time                    (カウンターレベル)
```

### チェックボックスの種類

#### 1. オブジェクトレベル（三状態チェックボックス）
- **全選択状態 (✅)**: 配下の全インスタンス・カウンターが選択済み
- **部分選択状態 (⚠️)**: 一部のインスタンス・カウンターが選択済み
- **全解除状態 (❌)**: 配下の全インスタンス・カウンターが未選択

#### 2. インスタンスレベル（三状態チェックボックス）  
- **全選択状態 (✅)**: 配下の全カウンターが選択済み
- **部分選択状態 (⚠️)**: 一部のカウンターが選択済み
- **全解除状態 (❌)**: 配下の全カウンターが未選択

#### 3. カウンターレベル（二状態チェックボックス）
- **選択状態 (✅)**: そのカウンターが選択済み
- **未選択状態 (❌)**: そのカウンターが未選択

## 使用方法

### オブジェクトレベルでの一括選択

**操作**: `Processor`オブジェクトのチェックボックスをクリック

**効果**:
- `Processor`配下の全インスタンス（`_Total`, `0`, `1`など）が選択される
- 各インスタンス配下の全カウンターが選択される
- 結果として、CPUに関する全ての監視項目が一度に選択される

**使用場面**: 
- システム全体のCPU性能を包括的に監視したい場合
- 初期調査でCPU関連の全指標を確認したい場合

### インスタンスレベルでの選択

**操作**: `_Total`インスタンスのチェックボックスをクリック

**効果**:
- `_Total`配下の全カウンター（`% Processor Time`, `% User Time`など）が選択される
- 親の`Processor`オブジェクトは部分選択状態になる（他インスタンスが未選択のため）

**使用場面**:
- 特定のCPUコアやプロセスの詳細を監視したい場合
- 全体値（_Total）のみで十分な場合

### カウンターレベルでの個別選択

**操作**: `% Processor Time`カウンターのチェックボックスをクリック

**効果**:
- そのカウンターのみが選択される
- 親の`_Total`インスタンスと`Processor`オブジェクトは部分選択状態になる

**使用場面**:
- 特定の指標のみを監視したい場合
- 関連する複数の指標を個別に選択したい場合

## 状態の自動同期

### 子から親への状態伝播

```
カウンター選択
↓
インスタンスの状態が自動更新
↓
オブジェクトの状態が自動更新
```

**例**: 
1. `% Processor Time`を選択
2. `_Total`が部分選択状態に自動変更
3. `Processor`が部分選択状態に自動変更

### 親から子への状態伝播

```
オブジェクト選択
↓
全インスタンスが自動選択
↓
全カウンターが自動選択
```

**例**:
1. `Processor`を選択
2. `_Total`, `0`, `1`などの全インスタンスが自動選択
3. 各インスタンス配下の全カウンターが自動選択

## 実用的な活用例

### シナリオ1: 初期調査
```
1. Processorオブジェクトを選択 → CPU関連の全指標を表示
2. Memoryオブジェクトを選択 → メモリ関連の全指標を表示
3. データテーブルとグラフで全体的な傾向を把握
```

### シナリオ2: 詳細分析
```
1. 初期調査で問題のあるオブジェクトを特定
2. 該当オブジェクトを解除
3. 特定のインスタンスまたはカウンターのみを選択
4. 詳細な分析を実施
```

### シナリオ3: 継続監視
```
1. 重要なカウンターのみを個別選択
2. 選択状態を維持してデータ収集
3. 定期的にグラフとデータを確認
```

## 技術仕様

### チェック状態の管理
- `bool?`型プロパティによる三状態管理
- `true`: 選択, `false`: 未選択, `null`: 部分選択

### 親子関係の実装
- `Parent`プロパティによる親ノードへの参照
- `Children`コレクションによる子ノードの管理

### 状態同期の仕組み
- プロパティ変更時の自動的な親子状態更新
- 再帰的な状態伝播による一貫性の維持

## トラブルシューティング

### よくある質問

**Q: 部分選択状態のチェックボックスをクリックするとどうなりますか？**
A: 部分選択状態から全選択状態に変わり、配下の全要素が選択されます。

**Q: 大量のカウンターを選択した場合のパフォーマンスは？**
A: 選択操作自体は高速ですが、データ読み込み（relog.exe実行）には時間がかかる場合があります。

**Q: 選択状態をリセットしたい場合は？**
A: 「すべて解除」ボタンをクリックするか、最上位のオブジェクトを手動で解除してください。

### パフォーマンス考慮事項

- **大量選択時**: オブジェクトレベルで選択すると数百のカウンターが同時選択される場合があります
- **メモリ使用量**: 選択されたカウンター数に比例してメモリ使用量が増加します
- **relog.exe実行時間**: 選択カウンター数とBLGファイルサイズに比例して処理時間が増加します
